# Pseudocode Seed

Use this skill to bootstrap pseudo-code coverage on a new project by focusing first on the hot files (highest git churn × lines-of-code).

## Steps

1. Call `pseudo_hot_files(project, limit=20)` to get the top-20 priority files.

2. Show the user the list with their current `file_prose_origin` status. Ask which files to seed — default to all files with `file_prose_origin = 'none'` or `'heuristic'`.

3. For each selected file:
   a. Call `pseudo_get_file_state(project, file_path)` to see the current methods and any existing heuristic prose.
   b. Read the source file.
   c. Draft manual prose per method using the existing `pseudocode` skill workflow.
   d. Call `pseudo_upsert_prose(project, { file, origin: 'llm', methods: [...] })` to commit.

4. After seeding, call `pseudo_db_status(project)` to show the new coverage breakdown.

## Budget

Respect a user-specified budget (files or tokens). Seeding 20 files in one pass is typical; seeding >50 should be broken into multiple runs.

## Notes

- Prose generated by this skill uses `origin: 'llm'` since the model is writing it.
- The overlay flow will auto-attach new prose to matching methods on the next scan.
- If you see orphan warnings afterward, run `pseudo_list_orphaned_prose` to clean them up.
