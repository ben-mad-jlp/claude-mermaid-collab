# Research — task `ws-browser-input`

Scope: `src/websocket/handler.ts`, `src/server.ts`. READ-ONLY findings for the IMPLEMENT node.

This task is **Change 3** of the blueprint (3a, 3b, 3c, 3e). NOTE: 3d (`canvasToPage`,
`BrowserFrameMeta`) and `dispatchMousePress`/`clickMouse`/`scrollMouse`/`dispatchKey` live in

## server.ts — exact change shapes

### 3c. `lastFrameMeta` cache
The sink is at **server.ts:211-222** inside the `if (screencastService)` block (203-235). Currently
the meta object is built INLINE as the `meta:` property of the `broadcastBrowserFrame` arg
(lines 214-220). The IMPLEMENT node must:
1. Add a module-level map. Put it near the other top-level `screencastUnsubscribers` map
   (**server.ts:202**), e.g. directly after line 202:
   ```ts
   const lastFrameMeta = new Map<string, import('./services/cdp-input').BrowserFrameMeta>();
   ```
   (Use the `import(...)` *type* form so no runtime import is added at module top; or add a top-level
   `import type { BrowserFrameMeta } from './services/cdp-input';`. Either works — type-only.)
2. Extract the inline `meta` literal into a named const inside the sink, then both broadcast and cache:
   ```ts
   const sink = (frame: { data: string; metadata: any; sessionName: string }) => {
     const meta = {
       offsetTop: frame.metadata.offsetTop,
       pageScaleFactor: frame.metadata.pageScaleFactor,
       deviceWidth: frame.metadata.deviceWidth,
       deviceHeight: frame.metadata.deviceHeight,
       timestamp: frame.metadata.timestamp,
     };
     lastFrameMeta.set(frame.sessionName, meta);
     wsHandler.broadcastBrowserFrame(frame.sessionName, { data: frame.data, meta });
   };
   ```
   NOTE: `meta` carries an extra `timestamp` field beyond `BrowserFrameMeta`'s 4 fields. Assigning a
   *named variable* (not an object literal) to `Map<string, BrowserFrameMeta>` is fine — TS excess-
   property checks only fire on fresh object literals, and a structural superset is assignable. ✔
   `canvasToPage` only reads the 4 geometric fields, so the extra `timestamp` is harmless.

### 3e. Register the handler — inside the SAME `if (screencastService)` block
Register `wsHandler.setBrowserInputHandler(...)` inside the `if (screencastService)` block (203-235),
e.g. right after the `setOnChannelSubscriptionChange(...)` call closes at line 234 but before the
block's closing `}` at 235. Use the blueprint's body (§3e) verbatim, with these confirmed facts:

- **CDP_PORT is NOT in module scope.** It is only destructured inside try-blocks at server.ts:60
  (`from './services/cdp-session.js'`) and server.ts:75 (`from './config.js'`). So the handler MUST
  obtain it locally. Recommended: `const { CDP_PORT } = await import('./config.js');` inside the
  handler callback (config.js is the canonical source; cdp-session re-imports from config too).
- `withCDPSession` is imported via `await import('./services/cdp-session.js')` — confirmed that's the
  module path used elsewhere (server.ts:60). It resolves session→target + throws friendly errors.
- `cdpInput` via `await import('./services/cdp-input.js')`.
- The handler is `async ({ session, event }) => { ... }` and the setter type is
  `(msg: { session: string; event: any }) => void` — an async arrow returning `Promise<void>` is
  assignable to a `=> void` callback (return value ignored). ✔ The `.catch(...)` on the
  `withCDPSession(...)` promise handles dispatch errors so nothing rejects unhandled.

Final handler body (CDP_PORT line added vs blueprint):
```ts
  wsHandler.setBrowserInputHandler(async ({ session, event }) => {
    const meta = lastFrameMeta.get(session);
    if (!meta && event.kind !== 'keydown' && event.kind !== 'keyup') return; // need meta for pointer coords
    const { CDP_PORT } = await import('./config.js');
    const { withCDPSession } = await import('./services/cdp-session.js');
    const cdpInput = await import('./services/cdp-input.js');
    await withCDPSession(session, CDP_PORT, async (client) => {
      switch (event.kind) {
        case 'mousemove': { const p = cdpInput.canvasToPage(meta!, event.nx, event.ny); await cdpInput.moveMouse(client, p.x, p.y); break; }
        case 'click':     { const p = cdpInput.canvasToPage(meta!, event.nx, event.ny); await cdpInput.clickMouse(client, p.x, p.y, event.button); break; }
        case 'mousedown': case 'mouseup': { const p = cdpInput.canvasToPage(meta!, event.nx, event.ny); await cdpInput.dispatchMousePress(client, event.kind, p.x, p.y, event.button); break; }
        case 'wheel':     { const p = cdpInput.canvasToPage(meta!, event.nx, event.ny); await cdpInput.scrollMouse(client, p.x, p.y, event.deltaX, event.deltaY); break; }
        case 'keydown':   await cdpInput.dispatchKey(client, 'keyDown', { key: event.key, code: event.code, text: event.text, modifiers: event.modifiers }); break;
        case 'keyup':     await cdpInput.dispatchKey(client, 'keyUp',   { key: event.key, code: event.code, modifiers: event.modifiers }); break;
      }
    }).catch((err) => console.error(`browser_input dispatch failed for ${session} —`, err));
  });
```

## Gotchas / confirmations
- `event` is typed `any` in the callback signature, so `event.kind`/`event.nx`/etc. need no narrowing
  in server.ts. ✔
- Do NOT add the handler outside the `if (screencastService)` block — panel input only makes sense in
  streamed-panel mode (blueprint §3e), and `lastFrameMeta` is only populated by the streamed sink.
- Horizontal scroll-offset is NOT in the L2 meta (no `scrollOffsetX/Y`); out of scope — do not invent.
- handler.ts has no CDP imports and must keep it that way (separation invariant). ✔
