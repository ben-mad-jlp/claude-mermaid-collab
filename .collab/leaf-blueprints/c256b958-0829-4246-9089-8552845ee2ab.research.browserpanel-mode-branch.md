## Risk notes

- **`streamed` dep in effect array:** Adding `streamed` to `[visible, activeId, streamed]` means the effect re-runs when mode resolves. On first render `mode === 'unknown'` → `streamed === false` → effect runs normally; once mode resolves to `'streamed'`, effect re-runs, hits the early return, and the previously-started rAF is cancelled via the cleanup return (the cleanup `cancelAnimationFrame(raf); bridge()?.setBounds?.(zero)` still executes from the prior run). This is safe.
- **`bridge()` is null in plain browser:** The rAF loop's `bridge()?.setBounds?.(rect)` already uses optional chaining, so even without the `if (streamed) return` guard it would be a no-op. The guard is still correct to avoid spinning a useless rAF loop.
- **`viewportRef` unused in streamed branch:** React does not error on unused refs; the `<div ref={viewportRef}>` simply never mounts in streamed mode. Keep the `useRef` declaration — removing it would break the native branch.
- **`currentSession.name` as channel key:** `Session.name` is the session identifier used in `subscriptionStore` keying (`${serverId}:${project}:${session}`); this matches what `broadcastBrowserFrame` uses server-side (`browser:<session>`).
