# Blueprint ‚Äî [L4] Streamed BrowserPanel: canvas renderer + mode detection

**Todo:** `c256b958-0829-4246-9089-8552845ee2ab`
**Epic:** Streamed Browser Panel (`streamed-panel` mode) ‚Äî see `docs/streamed-browser-panel-scope.md` ¬ß3.3.
**Sequencing:** after L2 (frame transport over WS ‚Äî `browser_frame` + `broadcastBrowserFrame`, `src/websocket/handler.ts:131,475`; per-session sink + `lastFrameMeta` in `src/server.ts:214`) and L3 (input plumbing). This leaf is the **UI consumer** of the L2 `browser:<session>` frame channel plus the server-side **mode flag** the UI needs to choose canvas-vs-placeholder.

## Goal

In `streamed-panel` mode, render the browser into a `<canvas>` fed by `browser_frame` WS messages instead of the native-overlay placeholder `<div>` at `ui/src/components/browser/BrowserPanel.tsx:261`, and **skip** the rAF `setBounds` loop (`BrowserPanel.tsx:61-106`) because there is no native `WebContentsView` to position. Mode is decided by a **server-reported capability flag**, never by sniffing `window.mc`. Per constraint **C3 (multi-server)**, the frame subscription binds to the **specific server connection** the session belongs to ‚Äî mirroring per-server artifact routing (`serverId` keying in `subscriptionStore`, `useServers()`), never the global singleton WS assumption.

Frame metadata (`offsetTop`, `pageScaleFactor`, `deviceWidth`, `deviceHeight`) is retained per-frame in a ref so **L5** (coordinate mapping for input) can consume it. This leaf does **not** wire input forwarding ‚Äî that is L5; it only renders and exposes the latest meta.

## Current state (verified)

- `BrowserPanel.tsx:6` `bridge()` = `window.mc?.browser` (native IPC). Tabs are native views populated via `browserStore.refresh()` ‚Üí `bridge().listTabs()`; in a plain browser tab `bridge()` is `null`, so `tabs` stays `[]`.
- `BrowserPanel.tsx:61-106` rAF effect calls `bridge().setBounds(rect)` to track the placeholder `<div ref={viewportRef} className="flex-1" />` (line 261). This is the **only** native-overlay coupling in the panel.
- L2 server already broadcasts on channel `browser:<session>`: `broadcastBrowserFrame` (`handler.ts:475-483`) ‚Üí `broadcastToChannel('browser:'+session, {type:'browser_frame', session, data, meta})`. Subscription is demand-driven (`server.ts:215-252`): screencast capture starts on first `browser:<session>` subscriber, stops on last unsubscribe.
- UI WS client: `WebSocketClient` (`ui/src/lib/websocket.ts:123`); `subscribe(channel)` sends `{type:'subscribe', channel}` (`:303`); `onMessage(handler)` returns an unsub (`:254`). Singleton via `getWebSocketClient(url)` (`:379`) keyed on a single default same-origin URL. The `WSMessage` union already carries `browser_frame` (added L2, `handler.ts:131`; the UI's mirror type lives in `ui/src/lib/websocket.ts`'s `WebSocketMessage`).
- Multi-server: `useServers()` (`ui/src/contexts/ServerContext.tsx:244`) returns `ServerInfo[]` (`{id,host,port,status,...}`). A `Session` carries `serverId` (`ui/src/types/session.ts:7`); `useSessionStore().currentSession` is the active one. `subscriptionStore` keys everything by `${serverId}:${project}:${session}`.
- Server mode constant: `MC_BROWSER_TARGET` (`src/config.ts:167`); `'streamed-panel'` already a recognized value (`config.ts:163`, `server.ts:71,84`). **No endpoint exposes it to the UI yet** ‚Äî this leaf adds one.
- Channel-subscribe wire detail: server stores subscriptions as `channel:browser:<session>` and counts viewers via `countChannelSubscribers` (`handler.ts:178,209`); the UI just calls `client.subscribe('browser:'+session)` ‚Äî do **not** prepend `channel:` (the server adds it).

### CREATE ‚Äî `ui/src/lib/serverFrameWs.ts`  *(C3: per-server WS resolver)*

A tiny resolver that returns the `WebSocketClient` bound to a given server, NOT a blind global singleton:

- `export function frameWsUrl(server?: { host: string; port: number }): string` ‚Äî when `server` is absent **or** its `host:port` equals the current `window.location.host`, return `getDefaultWebSocketURL()`-equivalent (same-origin `wss?://<host>/ws`). Otherwise build `${proto}//${server.host}:${server.port}/ws`.
- `export function getFrameClient(server?: { id?: string; host: string; port: number }): WebSocketClient` ‚Äî
  - Same-origin (or no server): return `getWebSocketClient()` (the existing singleton ‚Äî it already IS that server's connection in the plain-browser streamed-panel deployment, which is same-origin).
  - Cross-origin: lazily create + cache `new WebSocketClient(frameWsUrl(server))` in a `Map<string, WebSocketClient>` keyed by `server.id`; call `.connect()` once. Reuse on subsequent calls.
  - **Known gap (document inline, defer):** browser WS handshakes can't carry an `Authorization` header, so a *token-protected remote* server can't be reached this way directly. The `streamed-panel` deployment target (`docs/streamed-browser-panel-scope.md` ¬ß3.1) is a same-origin plain Chrome tab against its co-located server, so the same-origin path covers the shipping case; cross-origin auth is a follow-up. This is exactly the per-server scoping C3 requires ‚Äî the key point is we resolve by `serverId`/host:port, never assume one global stream source.

### CREATE ‚Äî `ui/src/hooks/useBrowserMode.ts`  *(mode detection, per-server)*

`export function useBrowserMode(server?: ServerInfo): 'streamed' | 'native' | 'unknown'`:

- Effect keyed on `server?.id` (+ host/port). Fetch the flag from the **session's own server**:
  - If `window.mc?.invokeOnServer` exists and `server` is set, use `mc.invokeOnServer(server.id, { path: '/api/browser/mode' })` (per-server fetch through the Electron main bridge ‚Äî mirrors `fetchCapabilities` in `ui/src/components/layout/SessionCard.tsx:23-30`).
  - Else (plain browser) `fetch('/api/browser/mode')` same-origin.
- Parse `{ streamed }`; set state to `'streamed'` when true, else `'native'`; `'unknown'` until resolved (treat as native for layout). Swallow errors ‚Üí `'native'` (native overlay is the safe default and never breaks the desktop app).
- Return the resolved value.

### CREATE ‚Äî `ui/src/components/browser/StreamedViewport.tsx`  *(the canvas renderer)*

`export function StreamedViewport({ session, server }: { session: string; server?: ServerInfo })`:

- `const canvasRef = useRef<HTMLCanvasElement>(null);`
- `const lastMetaRef = useRef<FrameMeta | null>(null);` where `FrameMeta = { offsetTop; pageScaleFactor; deviceWidth; deviceHeight; timestamp? }` (mirror of the L2 `browser_frame` `meta` shape). **Exported** (or via a `ref`/callback prop) so **L5** can read the latest meta for coord mapping. For this leaf, just keep it current.
- Effect keyed on `[session, server?.id]`:
  - `const client = getFrameClient(server);` (from `serverFrameWs.ts` ‚Äî **C3 binding**).
  - `await client.connect()` (idempotent), then `client.subscribe('browser:' + session)`.
  - `const unsub = client.onMessage((msg) => { if (msg.type !== 'browser_frame' || msg.session !== session) return; lastMetaRef.current = msg.meta; paint(msg.data); });`
  - Cleanup: `unsub(); client.unsubscribe('browser:' + session);` (drops the server-side viewer count ‚Üí screencast stops when last panel closes, per `server.ts:246`). Do **not** `disconnect()` the client (shared).
- `paint(dataB64)` ‚Äî MVP path: an offscreen `Image`; set `img.onload = () => { resize canvas to img.naturalWidth/Height; ctx.drawImage(img,0,0); }`; `img.src = 'data:image/jpeg;base64,' + dataB64`. (Optional faster path noted in comment: `createImageBitmap(blob)` + `drawImage` to avoid base64‚ÜíDOM decode; keep `<img>` for MVP per scope ¬ß3.3.) Reuse one `Image`/bitmap holder across frames; guard against painting after unmount.
- Markup: `<canvas ref={canvasRef} className="flex-1 w-full h-full object-contain bg-black" />`. No `setBounds`, no `viewportRef`.
- Render-rate note (comment only; coalescing is L5/P3): paint every frame for now ‚Äî CDP `screencastFrameAck` back-pressure (server side) already throttles producer rate.

## Risks / notes

- **C3 (multi-server):** the binding lives in `getFrameClient(server)` keyed by `serverId`/host:port ‚Äî verify it never falls back to the singleton for a *cross-origin* session (it returns a per-id cached client). The same-origin fast path is correct because the streamed-panel deployment is a plain Chrome tab served by the same server that streams.
- **Mode-fetch failure ‚Üí native:** any error resolving `/api/browser/mode` must degrade to the native placeholder so the desktop app is never broken by this change.
- **Viewer lifecycle:** unmount/unsubscribe must reach the server so `onChannelSubscriptionChange(count===0)` stops the screencast (`server.ts:246-251`) ‚Äî avoid leaking an idle capture. Don't `disconnect()` the shared client.
- **L5 boundary:** `lastMetaRef` is populated but no input listeners are attached here. Keep the meta shape identical to `browser_frame.meta` so L5 maps coords without translation drift.
- **Stale meta:** the ref reflects the most recent frame only; L5 should read it at event time, not cache it across resizes.
