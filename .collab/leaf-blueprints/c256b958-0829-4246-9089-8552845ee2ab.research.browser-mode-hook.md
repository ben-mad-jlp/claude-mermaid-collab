## Fetch strategy (per-server, mirrors existing patterns)

Two patterns in the codebase accomplish per-server fetches:

### Pattern A ‚Äî `mc.invokeOnServer` (Electron bridge), e.g. `useFleetStatus.ts:65-68`
```ts
const mc = (window as any).mc;
if (mc?.invokeOnServer) {
  const res = await mc.invokeOnServer(serverId, { path, method: 'GET' });
  // res.ok, res.body (object or string)
}
```
- `mc.invokeOnServer(serverId, { path, method })` routes the request through the Electron main-process bridge to the correct server process.
- Response shape: `{ ok: boolean, status?: number, body?: object | string, data?: any }`.

### Pattern B ‚Äî same-origin `fetch` (plain browser tab)
```ts
const r = await fetch('/api/browser/mode');
if (r.ok) data = await r.json();
```
- Used when `mc?.invokeOnServer` is absent (plain Chrome tab against same-origin server, which IS the streamed-panel deployment scenario).

### Decision logic for this hook (from blueprint):
- If `window.mc?.invokeOnServer` exists AND `server` is provided ‚Üí use `mc.invokeOnServer(server.id, { path: '/api/browser/mode', method: 'GET' })`.
- Else ‚Üí `fetch('/api/browser/mode')` same-origin.

Blueprint note: "mirrors `fetchCapabilities` in `ui/src/components/layout/SessionCard.tsx:23-30`" ‚Äî but `fetchCapabilities` uses `mc.getServerCapabilities`, NOT `invokeOnServer`. The actual closest pattern is `useFleetStatus.ts` and `useFleetStatusByProject.ts`, which use `mc.invokeOnServer` + same-origin fetch fallback. Use `useFleetStatus` as the direct model.

## Complete implementation sketch

```ts
import { useEffect, useState } from 'react';
import type { ServerInfo } from '@/contexts/ServerContext';

export function useBrowserMode(server?: ServerInfo): 'streamed' | 'native' | 'unknown' {
  const [mode, setMode] = useState<'streamed' | 'native' | 'unknown'>('unknown');

  useEffect(() => {
    let cancelled = false;
    const fetchMode = async () => {
      try {
        const mc = (window as any).mc;
        let streamed = false;
        if (mc?.invokeOnServer && server) {
          const res = await mc.invokeOnServer(server.id, { path: '/api/browser/mode', method: 'GET' });
          const body = res?.body ?? res?.data ?? res;
          const data = typeof body === 'string' ? JSON.parse(body) : body;
          streamed = data?.streamed === true;
        } else {
          const r = await fetch('/api/browser/mode');
          if (r.ok) {
            const data = await r.json();
            streamed = data?.streamed === true;
          }
        }
        if (!cancelled) setMode(streamed ? 'streamed' : 'native');
      } catch {
        if (!cancelled) setMode('native');
      }
    };
    void fetchMode();
    return () => { cancelled = true; };
  }, [server?.id, server?.host, server?.port]);

  return mode;
}
```

## Key constraints

1. **No `window.mc` sniffing for mode** ‚Äî always fetch `/api/browser/mode`; never infer from `!!window.mc`.
2. **Error ‚Üí `'native'`** ‚Äî any failure must fall back silently; must never break the desktop app.
3. **Effect keyed on `server?.id`** ‚Äî re-fetches if the active server changes (multi-server, C3).
4. **`'unknown'` until resolved** ‚Äî callers treat it as native; no flash of canvas before mode is confirmed.
5. **No polling** ‚Äî single fetch per server; mode is stable for a server's lifetime.
