## 1. Data contract (from P4a — verified, do NOT redefine the backend)

`GET /api/leaf-executor/stats?project=&epicId=&since=`
- Route: `src/routes/api.ts:2891`. Returns `getFleetStats({project?, epicId?, sinceTs?})`
  DIRECTLY (no `{ran}` envelope — unlike `/run/:leafId`). Always 200 with a `FleetStats`.
- All params optional. `since` is a numeric ms epoch (passed as `Number(since)`).

`FleetStats` (from `src/services/ledger-stats.ts:56`) — the EXACT shape the component types:

```ts
interface FleetStats {
  leafCount: number;
  nodesPerLeafAvg: number;          // float — render .toFixed(1)
  attemptRate: number;              // float, ~1.0–2.0 — .toFixed(2)
  blockRate: number;                // 0..1 fraction — render as %
  capPauseCount: number;            // int (rate-limited node rows)
  capPauseMs: number;               // Σ durationMs of rate-limited rows
  authModeAudit: Record<string, number>;  // count by authMode; healthy = {subscription: N}
  authModeAlarm: boolean;           // true iff any non-'subscription' key has count > 0
  wallClock: { p50: number; p90: number; max: number };  // per-leaf wallClockMs, ms
}
```

EMPTY state: when no rows match, `leafCount === 0`, all numeric fields `0`,
`authModeAudit === {}`, `authModeAlarm === false`, `wallClock` all `0`. The panel must
render a clean "No headless runs yet." empty state on `leafCount === 0` (NOT an alarm —
empty audit is not a violation).

## 2. Component: `ExecutorStatsPanel`

New file: `ui/src/components/supervisor/bridge/ExecutorStatsPanel.tsx`

Props:
```ts
{ project?: string; epicId?: string; serverScope?: string }
```
(Bridge passes `project`. `epicId` left undefined for the fleet view — optional scoping hook
for future drill-in; do NOT wire an epic selector in this leaf.) `serverScope` is accepted
but unused by the fetch (the API is server-local, same as WorkerRunStrip which omits it);
keep the prop for symmetry / future routing — or omit it. Implementer's call; omitting is fine.

### Fetch (MIRROR WorkerRunStrip.tsx:96–145 exactly — same cancelled-flag + nonce idiom)
- `useState`: `data: FleetStats | null`, `loading: boolean`, `refetchNonce: number`.
- Effect keyed `[project, epicId, refetchNonce]`: build URL with `URLSearchParams`
  (`project`, `epicId` only when set), `fetch(...).then(r => r.ok ? r.json() : null)`,
  guard with a `cancelled` flag (copy WorkerRunStrip:103–120). NOTE: response is the raw
  `FleetStats`, there is NO `ran` field — `setData(d)` directly.
- ws nudge (primary refresh): `getWebSocketClient().onMessage` → on
  `msg.type === 'session_todos_updated'` bump `refetchNonce`. Copy WorkerRunStrip:124–130
  verbatim. NO new event type.
- Bounded visible-only poll (fallback): a SLOW bounded interval (e.g. `POLL_MS = 15000`)
  that bumps the nonce, mounted only while the panel is rendered (the tab is the gate — it
  unmounts when another tab is active, so the interval is inherently "only while visible").
  Simpler/safer than WorkerRunStrip's terminal-gate since fleet stats have no terminal
  state. A manual refresh button in the header also bumps the nonce. Keep the cadence slow
  (≥15s) — this is evidence, not a live tail.

### Render — ONE card language (329741da)
Card chrome: copy WorkerRunStrip's outer shell —
`rounded-lg border border-gray-200 dark:border-gray-700 bg-gray-50/60 dark:bg-gray-900/40`,
a header row (`px-3 py-2 border-b ...`) with `text-[10px] uppercase tracking-wide
text-gray-400` label "executor stats" + a manual ↺ refresh button on the right
(`ml-auto`, same `text-2xs text-gray-400` weight).

HERO TILES (the type scale rule): each tile = a hero number `text-2xl tabular-nums` over a
label `text-xs uppercase tracking-wide text-gray-400`. Lay out as a responsive grid
(`grid grid-cols-2 sm:grid-cols-3 gap-3 px-3 py-3`). Five tiles, each `data-testid`:
| tile | value | testid |
|---|---|---|
| Leaves | `leafCount` | `stat-leafcount` |
| Nodes / leaf | `nodesPerLeafAvg.toFixed(1)` | `stat-nodesper` |
| Attempt rate | `attemptRate.toFixed(2)` | `stat-attemptrate` |
| Block rate | `(blockRate*100).toFixed(0) + '%'` | `stat-blockrate` |
| Cap pauses | `capPauseCount` | `stat-cappause` |

Hero numbers stay NEUTRAL (gray-900/gray-100). Block rate may go AMBER (not red) when
> 0 — amber is the "worth a look" tier; RED is reserved for the auth alarm only (329741da:
ONE red for genuine alarms). Keep block-rate amber optional/subtle; neutral is acceptable.

AUTH-MODE AUDIT row (the invariant made visible) — full-width band below the tiles,
`data-testid="authmode-audit"`:
- `authModeAlarm === false` (healthy, all subscription OR empty): GREEN band —
  `bg-green-100 text-green-700 dark:bg-green-900/40 dark:text-green-300`, label
  "auth: subscription-only ✓", show the `authModeAudit` breakdown (e.g. `subscription ×N`).
  Add `data-alarm="false"`.
- `authModeAlarm === true`: LOUD RED band —
  `bg-red-100 text-red-700 dark:bg-red-900/40 dark:text-red-300 font-bold`, label
  "AUTH ALARM — non-subscription auth detected", list the offending keys+counts from
  `authModeAudit` (every key !== 'subscription' with count > 0). Add `data-alarm="true"`.
  This is THE red in the panel.

WALL-CLOCK summary — a quiet single line (`text-2xs tabular-nums text-gray-500`,
`data-testid="wallclock-summary"`): `p50 / p90 / max` formatted via a `fmtDuration(ms)`
helper (copy WorkerRunStrip.tsx:54–58 — `<1000 → "Nms"`, else `"N.Ns"`; extend to minutes
if desired). e.g. `wall-clock  p50 42.0s · p90 1m10s · max 3m2s`.

EMPTY state (`!data || data.leafCount === 0`): below the header, a quiet
`text-2xs text-gray-400 italic` "No headless runs yet." — NO tiles, NO audit band, NO alarm.
(`loading && !data` → "loading…" in the header, same as WorkerRunStrip:179.)

`export const ExecutorStatsPanel` + `export default`.

## 5. Test plan — `ui/src/components/supervisor/bridge/ExecutorStatsPanel.test.tsx`

Mirror `WorkerRunStrip.test.tsx`: UI-only, no live executor. Mock `@/lib/websocket`
(`getWebSocketClient: () => ({ onMessage: () => ({ unsubscribe(){} }) })`) and `global.fetch`
via a `mockFetchOnce(body)` returning `{ ok:true, json: () => Promise.resolve(body) }`.

Fixtures:
- `HEALTHY`: `{ leafCount: 5, nodesPerLeafAvg: 3.2, attemptRate: 1.4, blockRate: 0.2,
  capPauseCount: 1, capPauseMs: 5000, authModeAudit: { subscription: 16 },
  authModeAlarm: false, wallClock: { p50: 42000, p90: 70000, max: 182000 } }`.
- `ALARM`: same but `authModeAudit: { subscription: 14, api: 2 }, authModeAlarm: true`.
- `EMPTY`: `{ leafCount: 0, nodesPerLeafAvg: 0, attemptRate: 0, blockRate: 0,
  capPauseCount: 0, capPauseMs: 0, authModeAudit: {}, authModeAlarm: false,
  wallClock: { p50: 0, p90: 0, max: 0 } }`.

Cases:
1. **Hero tiles render from mocked stats** (HEALTHY): `waitFor` then assert
   `getByTestId('stat-leafcount')` text `5`, `stat-nodesper` `3.2`, `stat-attemptrate` `1.40`,
   `stat-blockrate` `20%`, `stat-cappause` `1`. Assert `wallclock-summary` present.
2. **Auth audit GREEN when healthy**: `getByTestId('authmode-audit')` has
   `data-alarm="false"` and className contains a green token (e.g. `green-700` /
   `bg-green-100`); does NOT contain `red`.
3. **Auth audit LOUD RED when `authModeAlarm` true** (ALARM): `authmode-audit` has

4. **Empty / no-runs state** (EMPTY): assert "No headless runs yet." present and
