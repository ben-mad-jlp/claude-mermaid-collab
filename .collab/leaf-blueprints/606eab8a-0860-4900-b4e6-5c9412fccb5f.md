# Blueprint — [L3] Input plumbing: shared cdp-input.ts + browser_input WS

Leaf id: `606eab8a-0860-4900-b4e6-5c9412fccb5f`
Sequenced after L2 (both touch `src/websocket/handler.ts` and reuse the L2 frame
meta). Pure refactor of the MCP browser tools' Input calls into a shared module,
plus a new inbound `browser_input` WS path that drives the same module for the
live streamed panel.

## Goal

1. Factor the raw `Input.dispatchMouseEvent` / `Input.dispatchKeyEvent` logic out
   of `src/mcp/tools/browser.ts` into a new `src/services/cdp-input.ts` so the MCP
   tools **and** the live panel call ONE implementation. No behavioral change to
   any `browser_*` MCP tool — byte-identical CDP calls.
2. Add an inbound `browser_input` WS message (mouse / key / scroll) that resolves
   the session's CDP target and dispatches through `cdp-input.ts`.
3. Translate canvas-relative (normalized) coords → page coords using the L2 frame
   meta (`offsetTop`, `pageScaleFactor`, `deviceWidth`, `deviceHeight`) so clicks
   land where the user sees them.

## Reality grounding (what already exists)

- `src/mcp/tools/browser.ts` performs raw Input calls in exactly these spots:
  - `browserPressKey` (lines 161–167): `keyDown` + `keyUp` with `{ key }`.
  - `browserHover` (line 177): `mouseMoved` at `{ x, y }`.
  - `browserDrag` (lines 237–239): `mousePressed` → `mouseMoved` → `mouseReleased`
    (button `left`, `clickCount: 1`) at source/target box coords.
  - `browserTypeText` (lines 247–248): per-char `keyDown`/`keyUp` with `{ text, key }`.
  - (Note: `browserClick`/`browserFill`/`browserSelect` use `Runtime.evaluate` /
    `callFunctionOn`, NOT `Input.*` — they are OUT of scope, do not touch.)
- All tools run inside `withCDPSession(session, CDP_PORT, async (client) => …)`
  from `src/services/cdp-session.ts`. `client.Input` is the CDP Input domain.
- `withCDPSession` already resolves the session → targetId via the in-process
  `tabRegistry` and throws friendly errors if no tab/Chrome. Reuse it verbatim —
  do NOT add a second target-resolution path.
- L2 frame transport: `src/websocket/handler.ts` defines `browser_frame` with
  `meta: { offsetTop, pageScaleFactor, deviceWidth, deviceHeight, timestamp? }`
  (handler.ts:131-133) and `broadcastBrowserFrame()` (handler.ts:453-461). The
  sink in `src/server.ts` (lines 211-222) builds that meta from CDP
  `params.metadata`. The screencast client lives in `src/services/screencast.ts`.
- Inbound WS dispatch lives in `WebSocketHandler.handleMessage` (handler.ts:179).
  The existing decoupling pattern is `agentDispatcher` (a callback object set via
  `setAgentDispatcher`, handler.ts:149-151, invoked at 235-238). Mirror it for
  input so the handler stays free of any CDP import.

## Change 1 — NEW `src/services/cdp-input.ts`

Create a module of pure-ish dispatch helpers that take a CDP `client` (the object
yielded by `withCDPSession`) and perform the Input calls. Keep signatures minimal
and matched to current call sites so the refactor is provably behavior-preserving.

Exports (all `async`, all take `client: any` first — `any` matches the codebase's
untyped `chrome-remote-interface` usage):

```ts
// One key down+up. Mirrors browserPressKey (browser.ts:163-164).
export async function pressKey(client: any, key: string): Promise<void>

// Type a string char-by-char. Mirrors browserTypeText (browser.ts:247-248).
export async function typeText(client: any, text: string): Promise<void>

// Move the mouse. Mirrors browserHover (browser.ts:177).
export async function moveMouse(client: any, x: number, y: number): Promise<void>

// press → move → release. Mirrors browserDrag (browser.ts:237-239).
export async function dragMouse(client: any, sx: number, sy: number, tx: number, ty: number): Promise<void>

// NEW capabilities the live panel needs (not used by existing tools):
// full click = press+release at one point.
export async function clickMouse(client: any, x: number, y: number, button?: 'left'|'middle'|'right', clickCount?: number): Promise<void>
// wheel scroll at a point.
export async function scrollMouse(client: any, x: number, y: number, deltaX: number, deltaY: number): Promise<void>
// key down OR up individually (panel sends discrete keydown/keyup).
export async function dispatchKey(client: any, type: 'keyDown'|'keyUp'|'rawKeyDown'|'char', opts: { key?: string; text?: string; code?: string; windowsVirtualKeyCode?: number; modifiers?: number }): Promise<void>
```

Implementation = lift the exact bodies. e.g. `pressKey` is:
```ts
await client.Input.dispatchKeyEvent({ type: 'keyDown', key });
await client.Input.dispatchKeyEvent({ type: 'keyUp', key });
```
`dragMouse` lifts the three lines verbatim (same `button: 'left', clickCount: 1`).
`typeText` lifts the `for (const char of text)` loop verbatim. Do not "improve"
them — identical CDP params is the acceptance bar for the refactor.

## Change 2 — Refactor `src/mcp/tools/browser.ts` to call cdp-input

Add `import * as cdpInput from '../../services/cdp-input.js';` (match existing
`.js` extension import style on line 3).

- `browserPressKey`: replace the two `client.Input.dispatchKeyEvent` lines with

- `browserTypeText`: replace the loop body / loop with

- `browserHover`: replace the `client.Input.dispatchMouseEvent({ type:'mouseMoved', x, y })`

- `browserDrag`: replace the three Input calls with
  `await cdpInput.dragMouse(client, sx, sy, tx, ty);`.

Everything else in those functions (Runtime/DOM setup, box-model math, return

## Change 3 — `browser_input` inbound WS message + handler plumbing

### 3a. Wire type in `src/websocket/handler.ts`
Add to the `WSMessage` union (near the `browser_frame` member, ~line 131):
```ts
| { type: 'browser_input'; session: string; event:
    | { kind: 'mousemove'; nx: number; ny: number }
    | { kind: 'mousedown' | 'mouseup' | 'click'; nx: number; ny: number; button?: 'left'|'middle'|'right' }
    | { kind: 'wheel'; nx: number; ny: number; deltaX: number; deltaY: number }
    | { kind: 'keydown' | 'keyup'; key: string; code?: string; text?: string; modifiers?: number } }
```
`nx`/`ny` are normalized [0,1] coords relative to the displayed frame image (the
panel divides clientX/clientY by the canvas's rendered width/height before
sending — keeps the server independent of the panel's CSS size).

### 3b. Decoupling callback (mirror agentDispatcher)
Add to `WebSocketHandler`:
```ts
private browserInputHandler: ((msg: { session: string; event: any }) => void) | null = null;
setBrowserInputHandler(cb: (msg: { session: string; event: any }) => void): void { this.browserInputHandler = cb; }
```
In `handleMessage`, add a branch (after the `peer_registry` branch, ~line 258):
```ts
} else if (data.type === 'browser_input') {
  if (this.browserInputHandler) this.browserInputHandler({ session: data.session, event: data.event });
}
```
Keep handler.ts free of any CDP import — same separation the agentDispatcher uses.

### 3c. Latest-meta cache for coord translation
The translation needs the most recent frame meta per session. Cache it where the
sink already runs, in `src/server.ts`. Add a module-level
`const lastFrameMeta = new Map<string, BrowserFrameMeta>();` and, inside the
existing sink (server.ts:211-222), store the meta right before/after
`broadcastBrowserFrame`:
```ts
lastFrameMeta.set(frame.sessionName, meta);
```
(where `meta` is the object already being built). This avoids adding state to the
ScreencastService and reuses the exact meta L2 broadcasts.

### 3d. Translation helper (exported from cdp-input.ts for unit-testing)
```ts
export interface BrowserFrameMeta { offsetTop: number; pageScaleFactor: number; deviceWidth: number; deviceHeight: number }
// normalized [0,1] frame coords -> CDP page coords.
export function canvasToPage(meta: BrowserFrameMeta, nx: number, ny: number): { x: number; y: number } {
  const x = (nx * meta.deviceWidth) / (meta.pageScaleFactor || 1);
  const y = meta.offsetTop + (ny * meta.deviceHeight) / (meta.pageScaleFactor || 1);
  return { x, y };
}
```
Rationale: CDP `Page.screencastFrame` metadata reports the captured image as
`deviceWidth × deviceHeight` device px with page zoom `pageScaleFactor` and a top
visual-viewport offset `offsetTop`; dividing by `pageScaleFactor` returns CSS page
coords and adding `offsetTop` accounts for the captured viewport's vertical
offset. NOTE: horizontal/scroll offset is not in the L2 meta — if smooth-scroll
support is later needed, extend the L2 meta with `scrollOffsetX/Y`; out of scope
here (call out, don't silently drop).

### 3e. Register the handler in `src/server.ts`
Inside the existing `if (screencastService) { … }` block (server.ts:203-235) —
panel input only makes sense in streamed-panel mode — register:
```ts
wsHandler.setBrowserInputHandler(async ({ session, event }) => {
  const meta = lastFrameMeta.get(session);
  if (!meta && (event.kind !== 'keydown' && event.kind !== 'keyup')) return; // need meta for pointer coords
  const { withCDPSession } = await import('./services/cdp-session.js');
  const cdpInput = await import('./services/cdp-input.js');
  await withCDPSession(session, CDP_PORT, async (client) => {
    switch (event.kind) {
      case 'mousemove': { const p = cdpInput.canvasToPage(meta!, event.nx, event.ny); await cdpInput.moveMouse(client, p.x, p.y); break; }
      case 'click':     { const p = cdpInput.canvasToPage(meta!, event.nx, event.ny); await cdpInput.clickMouse(client, p.x, p.y, event.button); break; }
      case 'mousedown': case 'mouseup': { const p = cdpInput.canvasToPage(meta!, event.nx, event.ny); await cdpInput.dispatchMousePress(client, event.kind, p.x, p.y, event.button); break; }
      case 'wheel':     { const p = cdpInput.canvasToPage(meta!, event.nx, event.ny); await cdpInput.scrollMouse(client, p.x, p.y, event.deltaX, event.deltaY); break; }
      case 'keydown':   await cdpInput.dispatchKey(client, 'keyDown', { key: event.key, code: event.code, text: event.text, modifiers: event.modifiers }); break;
      case 'keyup':     await cdpInput.dispatchKey(client, 'keyUp',   { key: event.key, code: event.code, modifiers: event.modifiers }); break;
    }
  }).catch((err) => console.error(`browser_input dispatch failed for ${session} —`, err));
});
```
(Add a small `dispatchMousePress(client, kind, x, y, button)` helper to cdp-input —
maps `'mousedown'→'mousePressed'`, `'mouseup'→'mouseReleased'`, `clickCount:1`.)
`CDP_PORT` is already imported in server.ts via the cdp-session import chain;
otherwise import it alongside.
