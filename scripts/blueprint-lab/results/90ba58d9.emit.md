## Implementation Blueprint — HTTP transport mode for the pin smoke

### Goal
`scripts/smoke-conductor-pin-live.ts` today dispatches every route call **in-process** through `handleSupervisorRoutes` (line 51) and reaches into the **in-process store** for mission state (`getMission`, `setCriterionMet`) and the **in-process orchestrator** (`startOrchestrator`/`stopOrchestrator`). Add a second transport: when `SMOKE_HTTP_BASE_URL` is set, drive a **real deployed server** over `fetch`, read mission state via the HTTP missions GET route, and rely on the deployed server's own conductor timer instead of starting a local one.

### Design constraints discovered (must be honored)
- **No HTTP route sets a criterion verdict.** `supervisor-routes.ts:249-250` deliberately keeps met/unmet steward/MCP-only (maker≠checker). So Phase 4's convergence step (`setCriterionMet`, `scripts/smoke-conductor-pin-live.ts:163`) has no HTTP equivalent — it must be **gated to in-process mode** and clearly skipped-with-a-log in HTTP mode. See [[project_mission_ui_authoring]].
- **The missions GET route returns everything `sample()` needs.** `GET /api/supervisor/missions?project=…` → `{ missions: MissionSummary[] }` and `MissionSummary.mission` is a full `MissionRow` (`mission-store.ts:851`), so HTTP-mode `sample()` reads `lastConductorKey`/`lastNudgeAt`/`updatedAt`/`status` from the summary — no store import needed.
- **The deployed server runs its own conductor timer** (`orchestrator-live.ts:51`, `CONDUCTOR_INTERVAL_MS`). HTTP mode must NOT call `startOrchestrator()` — doing so would spin a second timer against the local isolated db. Skip it.

### Change shape — all edits in `scripts/smoke-conductor-pin-live.ts`

**1. Transport-selector constants** (add just after the imports, ~line 40, beside `CONDUCTOR_INTERVAL_MS`):
```ts
// HTTP transport mode: when SMOKE_HTTP_BASE_URL is set, every route call goes over
// real HTTP to a DEPLOYED server (e.g. http://localhost:9002) instead of the
// in-process handleSupervisorRoutes handler; mission state is read via the missions
// GET route and the deployed server's own conductor timer drives ticks.
const HTTP_BASE_URL = process.env.SMOKE_HTTP_BASE_URL?.replace(/\/+$/, '') ?? null;
const HTTP_MODE = HTTP_BASE_URL !== null;
```

**2. Transport-aware `route()`** (replace body at lines 50-55): in HTTP mode `fetch(req)` the real server; otherwise keep the in-process `handleSupervisorRoutes` dispatch:
```ts
async function route(req: Request): Promise<{ status: number; body: any }> {
  if (HTTP_MODE) {
    const res = await fetch(req);
    const body = await res.json().catch(() => null);
    return { status: res.status, body };
  }
  const res = await handleSupervisorRoutes(req, new URL(req.url));
  const status = res?.status ?? 0;
  const body = res ? await res.json().catch(() => null) : null;
  return { status, body };
}
```
