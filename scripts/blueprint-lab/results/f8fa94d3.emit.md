## Blueprint: Wire deterministic promotion into the conductor sweep

### Context (what exists)
`src/services/mission-store.ts:553` already defines `promoteQueuedMissions(project): string[]` — a pure, no-LLM primitive that, for every session with no active non-terminal mission, promotes the lowest-`queuePos` queued+approved+non-terminal mission to `active=1, queuePos=NULL`. Its companion `enqueueMission` (`mission-store.ts:522`) is what parks a mission with a FIFO `queuePos`. **Both primitives currently have ZERO callers** (verified: only self-reference in `mission-store.ts`), so a queued mission never becomes active — the conductor's `selectConductorMission` (`mission-store.ts:609`) only considers `m.mission.active` missions, so a queued mission is invisible to it forever.

This leaf wires the promotion primitive into the autonomous conductor sweep so that when a session's active mission goes terminal (converged/abandoned → `deactivateIfTerminal` drops its `active` flag), the next queued mission is promoted and driven in the very same pass.

### Where to wire it: `runConductorPassInner` (conductor-pass.ts)
The conductor pass is the right site (over reconcile-pass) because promotion feeds `selectConductorMission` with **zero latency** — promote, then select, in one tick — and promotion only matters when a driver (the conductor) actually exists. Reconcile-pass is throttled to 2.5 min and gated at level `on`, which would add latency and run promotion even when nothing drives.

Fail-open — a promotion write error must never sink the pass:

```ts
  if (!getConductorEnabled(project)) return { ran: false, reason: 'conductor-disabled' };

  // Deterministic FIFO promotion (no LLM): a session whose active mission just went
  // terminal (deactivateIfTerminal dropped its active flag) has no driver target until
  // its next queued mission is promoted. Run it HERE, before selection, so the promoted
  // mission is visible to selectConductorMission in this same pass (zero-latency handoff).
  // Fail-open: a promotion write must never break the conductor pass.
  try {
    const promoted = promoteQueuedMissions(project);
    if (promoted.length > 0) {
      console.warn(`[conductor] ${project}: promoted ${promoted.length} queued mission(s): ${promoted.join(', ')}`);
    }
  } catch (err) {
    console.warn(`[conductor] ${project}: queued-mission promotion failed:`, err instanceof Error ? err.message : err);
  }

  const pin = getConductorTargetMission(project);
```

This is project-wide (all sessions), independent of the pin, so a pinned run still promotes for other sessions. Since `promoteQueuedMissions` no-ops for any session that already has an active non-terminal mission, it is idempotent and cheap on the steady-state tick (one `listMissions` read, no writes).

```ts
test('promotes the next queued mission for a session whose active mission went terminal, then drives it', async () => {
  addWatchedProject(project);
  setConductorEnabled(project, true);
  // Mission A: active driver for session s1.
  const a = await forgeApprovedActive();
  // Mission B: same session → forged inactive (one-active-per-session); enqueue it FIFO.
  const b = await forgeMission(project, { session: 's1', title: 'B — a second capability', criteria: ['b is done'] });
  expect(getMission(project, b.missionId)?.active).toBe(false);
  enqueueMission(project, b.missionId);
  expect(getMission(project, b.missionId)?.queuePos).not.toBeNull();
  // A goes terminal → its active flag is cleared, so s1 now has no active non-terminal mission.
  setMissionAbandoned(project, a.missionId, Date.now());
  expect(getMission(project, a.missionId)?.active).toBe(false);
  // One conductor pass: promotes B (deterministic FIFO) AND drives it in the same pass.
  const r = await runConductorPass(project, { invoke: okInvoke });
  expect(r.ran).toBe(true);
  expect(r.reason).toBe('conducted');
  expect(r.missionId).toBe(b.missionId);
  const bAfter = getMission(project, b.missionId);
  expect(bAfter?.active).toBe(true);
  expect(bAfter?.queuePos).toBeNull(); // cleared on promotion
  expect(invokeCalls).toBe(1);
});
```
