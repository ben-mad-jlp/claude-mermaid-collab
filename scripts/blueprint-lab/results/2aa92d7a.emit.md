## Implementation Blueprint: GC orphan `collab/epic/*` branches (no live epic todo)

### Problem
`gcEpicBranches` (`src/services/landed-epic-sweep.ts:145`) iterates only `report.epics`, which `buildEpicBranchStatus` (`src/services/epic-branch-status.ts:130`) builds by walking **live epic todos** (`if (!isEpic(t)) continue;`). ### Change 1 — `BranchGcRunner` interface (`landed-epic-sweep.ts:88`)

Add two required methods after `deleteBranch`:

```ts
export interface BranchGcRunner {
  /** HEAD SHA of `branch`, or null if it doesn't resolve. */
  revParse(branch: string): string | null;
  /** Force-delete `branch` (git branch -D). Returns true on success. */
  deleteBranch(branch: string): boolean;
  /** All existing `collab/epic/<id8>` branch names (git for-each-ref). Empty on failure. */
  listEpicBranches(): string[];
  /** Commits `branch` carries beyond `baseRef` (git rev-list --count). Fail-closed:
   *  a probe error returns -1 (a NEGATIVE sentinel), so orphans with an unreadable
   *  ahead are FLAGGED, never deleted. Never returns null. */
  aheadCount(branch: string, baseRef: string): number;
}
```

### Change 2 — `makeBranchGcRunner` (`landed-epic-sweep.ts:113`)

Implement the two new methods using the existing `runGitLocal` helper (`landed-epic-sweep.ts:104`):

listEpicBranches(): string[] {
  const r = runGitLocal(project, ['for-each-ref', '--format=%(refname:short)', 'refs/heads/collab/epic/']);
  if (r.code !== 0) return [];
  return r.stdout.split('\n').map((s) => s.trim()).filter(Boolean);

aheadCount(branch: string, baseRef: string): number {
  const r = runGitLocal(project, ['rev-list', '--count', `${baseRef}..${branch}`]);
  if (r.code !== 0) return -1;
  const n = parseInt(r.stdout.trim() || '0', 10);
  return Number.isNaN(n) ? -1 : n;
},

### Change 3 — `gcEpicBranches` orphan pass (`landed-epic-sweep.ts:161-172`)

Introduce a `handled` set seeded by the epic-derived loop, then add an orphan pass. The `flagged` result field now also carries orphan **branch names** (no epic id exists for an orphan) — update its doc comment at `GcEpicBranchesResult.flagged` (`landed-epic-sweep.ts:98`) to say "epic ids (live pass) or branch names (orphan pass)".

```ts
  const deleted: string[] = [];
  const flagged: string[] = [];
  let skipped = 0;
  const handled = new Set<string>();

  for (const e of report.epics) {
    handled.add(e.branch);                       // dedup key for the orphan pass
    if (!e.exists) { skipped++; continue; }
    if ((e.ahead ?? 0) > 0) { flagged.push(e.epicId); continue; }
    const tip = runner.revParse(e.branch);
    if (tip == null) { skipped++; continue; }
    if (runner.deleteBranch(e.branch)) {
      appendRecoveryLog(project, e.branch, tip, now());
      deleted.push(e.branch);
    } else {
      skipped++;
    }
  }

  // Orphan pass: collab/epic/* refs with no live epic todo. Same fail-closed rule —
  // delete ONLY when git proves ahead===0; flag ahead>0 OR a probe error (ahead<0);
  // capture every deleted tip to the recovery log.
  for (const branch of runner.listEpicBranches()) {
    if (handled.has(branch)) continue;           // already processed via its live epic
    handled.add(branch);
    const ahead = runner.aheadCount(branch, baseRef);
    if (ahead !== 0) { flagged.push(branch); continue; }
    const tip = runner.revParse(branch);
    if (tip == null) { skipped++; continue; }
    if (runner.deleteBranch(branch)) {
      appendRecoveryLog(project, branch, tip, now());
      deleted.push(branch);
    } else {
      skipped++;
    }
  }

  return { deleted, flagged, skipped };
```

Note `ahead !== 0` folds both `ahead>0` and the `-1` probe-error sentinel into the flag branch, satisfying "flag ahead>0 (or a probe error, ahead<0)".
