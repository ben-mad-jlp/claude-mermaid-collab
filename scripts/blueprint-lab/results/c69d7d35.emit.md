## Blueprint: Port file written on bind; hooks + CLI read it and fail closed

### Problem & context

A sibling leaf in this epic (commit `2ccfa116`, "Single resolvedPort() + config.json port key") already unified the **configured** port into `getConfiguredPort()` (`src/services/config-file.ts:181`), reading `env PORT → config.json 'port' → 9002`. But two gaps remain:

1. **The actual bound port is not recorded anywhere a shell script can trivially read.** `writeLock()` records `port` into `$XDG_RUNTIME_DIR/mermaid-collab/server.lock` (`src/services/port-ownership.ts:164`) and `writeInstance()` records it into a per-session-hash discovery file — neither is a stable, plain path a hook can `cat`. When the server binds an OS-assigned ephemeral port (`PORT=0`, `PORT_REQUEST` at `src/config.ts:81`) the real port is knowable *only* after `Bun.serve()` returns (`src/server.ts:695` `const actualPort = server.port`).

2. **Every shell hook + the CLI hardcode `9002`.** Confirmed sites: `hooks/server-check.sh:14`, `scripts/statusline.sh:27,37`, `scripts/permission-hook.sh:73`, `scripts/notification-hook.sh:62`, `scripts/active-hook.sh:54`, `scripts/permission-denied-hook.sh:63`, `scripts/mc-port-handshake.sh:28`, and `bin/mermaid-collab.ts:31` (`process.env.PORT || 9002`). If the real server lives on a config-set or ephemeral port, these silently POST session telemetry to — or start a competitor against — whatever squats on `9002`.

**Fix:** write the actual bound port to a stable, shell-readable file (`~/.mermaid-collab/port`) on bind; clear it on shutdown. Give the hooks + CLI one authoritative resolver that prefers that file, so they target the *real* live server (fail closed onto the authoritative port) instead of a hardcoded guess.

### Change shape

#### 1. `src/services/config-file.ts` — the bound-port file layer (new)

Add, after the port-resolution block (below `setConfiguredPort`, `config-file.ts:203`):

```ts
import { unlinkSync } from 'node:fs';   // extend existing line-1 fs import

/** Path to the bound-port file: alongside config.json (~/.mermaid-collab/port),
 *  overridable via MERMAID_PORT_FILE. Following configPath()'s dir means tests that
 *  set MERMAID_CONFIG_PATH get an isolated port file for free. */
export function boundPortFilePath(): string {
  return process.env.MERMAID_PORT_FILE ?? join(dirname(configPath()), 'port');
}

/** Record the ACTUAL bound port on a successful bind (server.ts). Atomic (tmp+rename),
 *  best-effort dir create. This is the single shell-readable source of the live port. */
export function writeBoundPort(port: number): void {
  const p = boundPortFilePath();
  mkdirSync(dirname(p), { recursive: true });
  const tmp = p + '.tmp';
  writeFileSync(tmp, String(port) + '\n', 'utf8');
  renameSync(tmp, p);
}

/** Read the live bound port, or null if the file is absent/unparseable (fail closed). */
export function readBoundPort(): number | null {
  try {
    const raw = readFileSync(boundPortFilePath(), 'utf8').trim();
    const n = Number.parseInt(raw, 10);
    return Number.isNaN(n) || n < 1 || n > 65535 ? null : n;
  } catch {
    return null;
  }
}

/** Remove the bound-port file on shutdown so a dead server leaves no stale port behind. */
export function clearBoundPort(): void {
  try { unlinkSync(boundPortFilePath()); } catch { /* best-effort */ }
}
```

(`configPath`, `mkdirSync`, `writeFileSync`, `renameSync`, `readFileSync`, `dirname`, `join` are all already imported/defined in this file.)

#### 4. `scripts/mc-resolve-port.sh` — shared shell resolver (new file)

One `mc_resolve_port` function, sourced by every shell consumer so the precedence lives in exactly one place:

```bash
#!/usr/bin/env bash
# Single source of the port a hook/CLI should talk to. Precedence, fail-closed
# onto the AUTHORITATIVE live port rather than a hardcoded 9002:
#   $MERMAID_PORT → $PORT → bound-port file (live server) → config.json 'port' → 9002
mc_resolve_port() {
  if [ -n "${MERMAID_PORT:-}" ]; then echo "$MERMAID_PORT"; return; fi
  if [ -n "${PORT:-}" ];         then echo "$PORT";         return; fi
  local pf="${MERMAID_PORT_FILE:-${HOME}/.mermaid-collab/port}"
  if [ -r "$pf" ]; then
    local p; p="$(tr -dc '0-9' < "$pf")"
    if [ -n "$p" ]; then echo "$p"; return; fi
  fi
  local cf="${MERMAID_CONFIG_PATH:-${HOME}/.mermaid-collab/config.json}"
  if command -v jq >/dev/null 2>&1 && [ -r "$cf" ]; then
    local p; p="$(jq -r '.port // empty' "$cf" 2>/dev/null)"
    if [ -n "$p" ]; then echo "$p"; return; fi
  fi
  echo 9002
}
```

#### 5. Wire the resolver into each shell consumer

In each script below, source the helper relative to the script's own dir (robust against cwd), then set `PORT`/`MERMAID_PORT` from `mc_resolve_port` and substitute it into the hardcoded `:9002` URL. Pattern (adjust the relative path per script):

```bash
_D="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
# shellcheck source=/dev/null
. "${_D}/mc-resolve-port.sh" 2>/dev/null || mc_resolve_port() { echo "${MERMAID_PORT:-9002}"; }
PORT="$(mc_resolve_port)"
```

- `hooks/server-check.sh:14` — replace `PORT=${MERMAID_PORT:-9002}` with the sourced resolver (source path `../scripts/mc-resolve-port.sh`). `PORT` then flows through `check_server`/`wait_for_server` and the `bun run src/server.ts` spawn unchanged.
- `scripts/statusline.sh:27,37` — resolve once at top; change both `http://localhost:9002/...` to `http://localhost:${PORT}/...`.
- `scripts/permission-hook.sh:73`, `scripts/notification-hook.sh:62`, `scripts/active-hook.sh:54`, `scripts/permission-denied-hook.sh:63` — same one-line resolve + `:${PORT}` substitution in each POST URL.
- `scripts/mc-port-handshake.sh:28` — replace `PORT="${MC_PORT:-${PORT:-9002}}"` with `MC_PORT`/`PORT`-first then `mc_resolve_port` (so the systemd preflight guards the config-set port, not a hardcoded one).

### Out of scope / notes
- Multi-instance: the port file is a single global path representing the canonical server (mirrors the single global `server.lock`); per-project secondary instances on non-canonical ports also write it best-effort — full multi-instance disambiguation stays with the existing per-session instance-discovery files and is not addressed here.
- Shell scripts keep a `mc_resolve_port` fallback stub if the source fails, so a packaging mishap degrades to today's `9002` default rather than breaking the hook.
