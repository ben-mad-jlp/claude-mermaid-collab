# Blueprint — Make `emit.ts` run at the daemon's real blueprint model and persist failure diagnostics

## Context / problem

`scripts/blueprint-lab/emit.ts` is the lab harness that re-runs the real blueprint-node prompt against a git-mined corpus (`corpus.ts`) and checks whether the reply parses into a v2 `DiffContract`. Two defects:

1. **Wrong model.** Its header comment (`emit.ts:1-11`) promises it runs "at the model/effort the daemon actually runs (NOT the opus/high NODE_PROFILE.blueprint default)", but `buildSpec` (`emit.ts:58-67`) hardcodes `process.env.BLUEPRINT_MODEL || 'sonnet'` and `process.env.BLUEPRINT_EFFORT || 'medium'`. Those literals are a *guess*. The daemon's actual blueprint model/effort come from the same resolution the leaf-executor uses (`leaf-executor.ts:1747-1759`): `resolveNodeModel(project, 'blueprint', resolveNodeProvider(project,'blueprint',bpTools), NODE_PROFILE.blueprint.model)` for the model, and `nodeOverrides.blueprint?.effort ?? projectEffort ?? ENV_NODE_EFFORT ?? NODE_PROFILE.blueprint.effort` for effort — driven by the project's `node_profile_override` / `orchestrator_config` DB rows. So the lab measures a model the daemon may not run (memory: *benchmark daemon nodes at the model the daemon RUNS via node_profile_override, not the default*).

2. **Failures are un-diagnosable.** When a node yields no result, `runEmitNode` collapses everything into `text = [[NO RESULT]] stderr=${err.slice(-400)}` (`emit.ts:91`), discarding exit code, full stderr, and the argv. When `parseDiffContract` returns `null` (`emit.ts:117`) only the id lands in `run.json`'s `unparsed` list — there is nothing on disk to explain *why* the case failed.

The daemon keys this repo's config by `MERMAID_PROJECT`, which defaults to `process.cwd()` (`config.ts:90`) = the repo root. The lab's `REPO_ROOT` (`emit.ts:25`) is that same path, so it is the correct project key for the DB lookups.

The header comment's non-goal list ("does not modify diff-contract.ts / leaf-executor.ts / node-invoker.ts / corpus.ts") stays true — we only *import* from those; the only file edited is `emit.ts`.

### 2. Resolve the daemon's real blueprint model + effort

Add, after `REPO_ROOT` (`emit.ts:25`):

```ts
/** Project key the daemon uses for THIS repo — MERMAID_PROJECT defaults to cwd = repo root
 *  (src/config.ts). node_profile_override / orchestrator_config rows are keyed on it. */
const PROJECT = process.env.MERMAID_PROJECT || REPO_ROOT;

/** The model + effort the DAEMON actually runs the blueprint node at — the same resolution
 *  the leaf-executor uses (leaf-executor.ts nodeModel/nodeEffort), NOT the opus/high
 *  NODE_PROFILE.blueprint default and NOT a hardcoded sonnet/medium guess. BLUEPRINT_MODEL /
 *  BLUEPRINT_EFFORT stay as explicit experiment overrides (top precedence). DB reads are
 *  wrapped so a missing/locked .collab DB falls back to the profile default instead of
 *  crashing the lab. */
function resolveBlueprintModelEffort(): { model: string; effort: EffortLevel } {
  const bpTools = NODE_PROFILE.blueprint.allowedTools;
  let model = NODE_PROFILE.blueprint.model;
  let effort: EffortLevel = NODE_PROFILE.blueprint.effort;
  try {
    const provider = resolveNodeProvider(PROJECT, 'blueprint', bpTools);
    model = resolveNodeModel(PROJECT, 'blueprint', provider, NODE_PROFILE.blueprint.model);
    effort = listNodeProfileOverrides(PROJECT).blueprint?.effort
      ?? getProjectEffort(PROJECT)
      ?? NODE_PROFILE.blueprint.effort;
  } catch { /* no DB → keep the NODE_PROFILE.blueprint defaults */ }
  return {
    model: process.env.BLUEPRINT_MODEL || model,
    effort: (process.env.BLUEPRINT_EFFORT as EffortLevel) || effort,
  };
}

const BLUEPRINT = resolveBlueprintModelEffort();
```

### 4. `runEmitNode` returns full diagnostics (`emit.ts:69-97`)

Introduce a `NodeRun` return shape and thread `spec` in (so the argv/model/effort are recoverable):

```ts
interface NodeRun { text: string; raw: string; stderr: string; exitCode: number | null; argv: string[]; }

function runEmitNode(cwd: string, spec: ReturnType<typeof buildSpec>): Promise<NodeRun> {
  const argv = buildNodeArgv(spec as any);
  return new Promise((resolve) => {
    const child = spawn(argv[0], argv.slice(1), { cwd, stdio: ['pipe','pipe','pipe'], env: { ...process.env } });
    let out = ''; let err = '';
    child.stdout.on('data', (d) => (out += d));
    child.stderr.on('data', (d) => (err += d));
    child.on('close', (code) => {
      let text = '';
      for (const line of out.split('\n')) {
        const t = line.trim();
        if (!t.startsWith('{')) continue;
        try { const o = JSON.parse(t); if (o.type === 'result' && typeof o.result === 'string') text = o.result; }
        catch { /* skip non-json lines */ }
      }
      if (!text) text = `[[NO RESULT]] stderr=${err.slice(-400)}`;
      resolve({ text, raw: out, stderr: err, exitCode: code, argv });
    });
    child.stdin.write(spec.prompt);
    child.stdin.end();
  });
}
```

### 5. Persist per-case failure diagnostics (`runOne`, `emit.ts:113-121`)

Add the diagnostics type and write a `<id>.emit.diag.json` for every case whose contract fails to parse. Add a `diagnostics` field to `EmitResult` (`emit.ts:106-111`):

```ts
interface EmitDiagnostics {
  id: string; model: string; effort: EffortLevel;
  exitCode: number | null; argv: string[]; stderr: string;
  stdoutTail: string; textTail: string;
}

interface EmitResult {
  id: string;
  leafKindExpected: CorpusCase['leafKind'];
  contract: DiffContract | null;
  rawText: string;
  diagnostics: EmitDiagnostics | null;
}

async function runOne(c: CorpusCase): Promise<EmitResult> {
  const cwd = checkoutBase(c);
  const prompt = buildV2BlueprintPrompt(c);
  const spec = buildSpec(prompt);
  const run = await runEmitNode(cwd, spec);
  const contract = parseDiffContract(run.text);
  writeFileSync(join(OUT, `${c.id}.emit.md`), run.text);
  let diagnostics: EmitDiagnostics | null = null;
  if (contract === null) {
    diagnostics = {
      id: c.id, model: spec.model, effort: spec.effort,
      exitCode: run.exitCode, argv: run.argv, stderr: run.stderr,
      stdoutTail: run.raw.slice(-2000), textTail: run.text.slice(-2000),
    };
    writeFileSync(join(OUT, `${c.id}.emit.diag.json`), JSON.stringify(diagnostics, null, 2));
  }
  try { rmSync(cwd, { recursive: true, force: true }); } catch {}
  return { id: c.id, leafKindExpected: c.leafKind, contract, rawText: run.text, diagnostics };
}
```
