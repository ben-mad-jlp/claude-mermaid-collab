## Implementation Blueprint — Archive storage layer

### Goal
Add the persistence primitive for todo archiving to `src/services/todo-store.ts`: a nullable `archivedAt` timestamp column, partial indexes scoped to live (non-archived) rows, and archive-excluding filters on the two hot read paths (`listTodos` / `listTodosChunked`), gated by a new `TodoFilter.includeArchived` flag that mirrors the existing `includeCompleted` flag. No archive *writer* ships in this leaf — the column defaults NULL, so every runtime read is inert on current data (all rows read `archivedAt IS NULL`).

This follows the exact additive-column pattern already used for `landedAt` (W3) and the partial-index pattern used for `idx_todos_bucket_singleton`.

### Changes — all in `src/services/todo-store.ts`

**1. Column in `CREATE TABLE` DDL** (`DDL` const, after `landedAt TEXT,` at `todo-store.ts:509`):
```
  archivedAt TEXT,
```
This gives new DBs the column at create time (grouped with the other lifecycle timestamps `completedAt`/`landedAt`).

**2. `addColumnIfMissing` for existing DBs** (in `openDb`, immediately after the `landedAt` ALTER at `todo-store.ts:621`):
```ts
  // Archive storage layer: nullable soft-archive stamp. Additive; pre-migration rows and
  // every create read NULL (never inserted on the create path) → todo stays in the live
  // working graph until an archive writer stamps this. rowToTodo coalesces null → null.
  addColumnIfMissing(db, 'todos', 'archivedAt', 'archivedAt TEXT');
```
This is the ALTER path by which the column reaches an already-created `todos` table (existing DBs never re-run `CREATE TABLE`), mirroring the `landedAt` comment at `todo-store.ts:619-621`.

**3. Partial indexes on live rows** — created *after* all `addColumnIfMissing` calls (immediately after the `reservedReason` ALTER at `todo-store.ts:633`, before the one-shot backfills at `:636`). They MUST come after the column ALTER — a `WHERE archivedAt IS NULL` index in the `DDL` string would throw on an existing DB where the column doesn't exist yet at DDL-exec time (same ordering constraint that puts `idx_todos_bucket_singleton` inside its backfill at `todo-store.ts:876`):
```ts
  // Archive-excluding hot reads: partial indexes covering ONLY live rows (archivedAt IS
  // NULL). They keep the index small at scale (the ~80k-row scan cited in listTodosChunked)
  // and directly serve the default `archivedAt IS NULL` predicate the hot reads now add.
  // IF NOT EXISTS → idempotent on every open (same discipline as idx_todos_status at :533).
  db.exec(`CREATE INDEX IF NOT EXISTS idx_todos_live_status ON todos(status) WHERE archivedAt IS NULL`);
  db.exec(`CREATE INDEX IF NOT EXISTS idx_todos_live_owner ON todos(ownerSession) WHERE archivedAt IS NULL`);
```

**4. `TodoRow` interface** (after `landedAt: string | null;` at `todo-store.ts:470`):
```ts
  archivedAt: string | null;
```

**5. `Todo` interface** (after the `landedAt` block ending at `todo-store.ts:125`):
```ts
  /** Archive storage layer: ISO stamp of when this todo was soft-archived out of the live
   *  working graph, or null while live. Excluded by default from the hot reads (listTodos /
   *  listTodosChunked) unless TodoFilter.includeArchived is set. Optional so existing Todo
   *  literals / fixtures that predate the column still typecheck (rowToTodo always sets it). */
  archivedAt?: string | null;
```

**6. `rowToTodo` mapping** (after `landedAt: row.landedAt ?? null,` at `todo-store.ts:1305`):
```ts
    archivedAt: row.archivedAt ?? null,
```

**7. `TodoFilter.includeArchived`** (in `TodoFilter`, after `includeCompleted?: boolean;` at `todo-store.ts:192`):
```ts
  /** When false/absent, the hot reads exclude soft-archived rows (archivedAt IS NULL).
   *  Set true to include archived rows (audit / history views). Mirrors includeCompleted. */
  includeArchived?: boolean;
```

**8. Archive-excluding predicate in `listTodos`** (after the `includeCompleted` push at `todo-store.ts:1367`):
```ts
  if (!filter.includeArchived) { where.push('archivedAt IS NULL'); }
```

**9. Archive-excluding predicate in `listTodosChunked`** (after the `includeCompleted` push at `todo-store.ts:1405`):
```ts
  if (!filter.includeArchived) { where.push('archivedAt IS NULL'); }
```
The predicate is added to the shared `where` array before keyset pagination begins, so it applies uniformly across every page and preserves the documented result-parity contract with `listTodos` (`todo-store.ts:1385-1389`).

### New test — `src/services/__tests__/todo-archive-column.test.ts`
`bun test` style (touches `bun:sqlite`), mirroring `todo-bucket-column.test.ts`. Named tests:
- **`adds archivedAt column on open of a pre-column DB`** — seed a raw `todos` table WITHOUT `archivedAt` + `user_version=0`, `openDb`, assert `PRAGMA table_info(todos)` contains `archivedAt`.
- **`creates the two live partial indexes`** — after `openDb`, assert `SELECT COUNT(*) FROM sqlite_master WHERE type='index' AND name LIKE 'idx_todos_live_%'` returns `2`, and that each index's `sql` contains `WHERE archivedAt IS NULL`.
- **`listTodos excludes archived rows by default and includes them with includeArchived`** — `createTodo` two live todos, `UPDATE todos SET archivedAt=? WHERE id=?` one of them via `openDb`, assert `listTodos(project,{includeCompleted:true})` omits the archived id and `listTodos(project,{includeCompleted:true,includeArchived:true})` includes it.
- **`listTodosChunked excludes archived rows by default`** — same seeding, assert `await listTodosChunked(project,{includeCompleted:true},{pageSize:1})` omits the archived id (small pageSize to exercise pagination).
- **`create path leaves archivedAt null`** — `createTodo(...)` then `getTodo`, assert `archivedAt` is `null`.

### Acceptance criteria (positive, citable)
1. `archivedAt TEXT` column declared in the `DDL` `CREATE TABLE` block — `src/services/todo-store.ts` (`DDL`).
2. `addColumnIfMissing(db, 'todos', 'archivedAt', 'archivedAt TEXT')` present in `openDb` — `src/services/todo-store.ts`.
3. Two partial indexes `idx_todos_live_status` and `idx_todos_live_owner`, each `WHERE archivedAt IS NULL`, created via `db.exec` in `openDb`.
4. `archivedAt` field on both the `TodoRow` and `Todo` interfaces, and mapped in `rowToTodo`.
5. `includeArchived` field on `TodoFilter`; `!filter.includeArchived` pushes `archivedAt IS NULL` in BOTH `listTodos` and `listTodosChunked`.
6. New test file `src/services/__tests__/todo-archive-column.test.ts` with the named tests above, all green under `bun test`.

### Out of scope (noted, not criteria)
- The archive *writer* / MCP verb (`setArchived` / `archive_todo`) — a follow-up leaf. This leaf ships only the column, indexes, and read exclusion; because nothing stamps `archivedAt` yet, every runtime read is inert (all rows `archivedAt IS NULL`), which is the safety property that makes this landable alone.
- Backfilling existing `done`/`dropped` rows to archived — deliberately not done; archiving policy is a separate decision.
- No `user_version` bump: index creation is `IF NOT EXISTS` (idempotent catalog check, like `idx_todos_status`), and the column ALTER is guarded by `addColumnIfMissing` — neither needs a one-shot backfill gate.
