## Blueprint: Make `live-conductor-pin-evidence.ts` scratch-scoped and tear down its missions

### Problem
`scripts/live-conductor-pin-evidence.ts` today targets `--project=<path> ?? process.cwd()` (lines 28–30) — an operator's **real** git repo. It creates+approves missions A, B, and a terminal mission C (lines 138–148, 150–154, 217) and **never deletes any of them or unregisters the project**. Running the harness permanently litters the chosen project with 3+ throwaway missions. The sibling `scripts/smoke-conductor-pin-live.ts` already solves this: throwaway scratch git repo (`mkdtempSync` + `git init`), `isSelfProject` guard, and a `finally` teardown (smoke lines 235, 246–250, 411–417). This leaf brings the evidence harness up to the same discipline — while keeping it HTTP-only against the already-running server.

```ts
async function del(path: string, body: unknown): Promise<{ status: number; body: any }> {
  const res = await fetch(`${BASE_URL}${path}`, {
    method: 'DELETE',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(body),
  });
  return { status: res.status, body: await res.json().catch(() => null) };
}
```

**6. Wrap the pin/poll/assert body in `try` + a `finally` teardown.** Enclose everything from Step (a) (line 124) through the evidence write + verdict logging (currently lines 238–256) so teardown always runs. Critically, **move `process.exit` out of the `try`** — `process.exit` inside a `try` skips `finally`. Structure:
```ts
let verdict: 'PASS' | 'FAIL' = 'FAIL';
try {
  // Steps (a)–(f) … compute verdict, write evidence blob, log summary
} finally {
  log('\nCleanup');
  const unpin = await post('/api/supervisor/conductor', { project, targetMissionId: null }).catch(() => null);
  void unpin;
  for (const todoId of createdMissions) {
    const d = await del('/api/supervisor/missions', { project, todoId }).catch(() => null);
    log(`  🧹 deleted mission ${todoId}${d?.body?.deleted === todoId ? '' : ' (best-effort)'}`);
  }
  await del('/api/supervisor/projects', { project }).catch(() => {});
  if (existsSync(project)) { rmSync(project, { recursive: true, force: true }); log(`  🧹 removed scratch project`); }
}
process.exit(verdict === 'PASS' ? 0 : 1);
```
`DELETE /api/supervisor/missions` routes to `delete_mission` and returns `{ deleted: todoId }` (`src/routes/supervisor-routes.ts:281-290`, `src/mcp/mission-tools.ts:178-197`); `DELETE /api/supervisor/projects` unwatches + unregisters (`src/routes/supervisor-routes.ts:395-406`). Both are the exact routes `smoke-conductor-pin-live.ts` relies on. Assign the computed verdict to the outer `verdict` var inside the `try` (replacing the local `const verdict` at line 241 with an assignment) so the post-`finally` `process.exit` sees it.

### Notes on correctness
- Teardown is best-effort (`.catch`) so a delete failure never masks the real PASS/FAIL verdict, but each deletion is logged so a silent leak is visible.
- The scratch repo is a bare `git init` — the conductor's actionability logic only needs registered+approved missions, which the harness creates over HTTP against the running server (same as smoke live mode, which registers its scratch repo with the deployed server at smoke lines 252–253).
