## Blueprint — Live-evidence harness: two consecutive sweeps over the self/live project

### What exists today

`runSweepMeasurement(project, opts)` (`src/services/sweep-measurement.ts:55`) is a composed, read-mostly sweep (GC is its only mutation) that returns a `SweepMeasurement` snapshot (`src/services/sweep-measurement.ts:33-41`) whose module header explicitly states callers should "diff across two consecutive runs to test idempotence" (`sweep-measurement.ts:6`). **It currently has zero callers and zero tests** (confirmed: only reference is its own definition). Nothing yet performs the two-consecutive-run diff the primitive was built for, and nothing runs it against the real self/live project.

This leaf builds that missing consumer as **live evidence**: an in-process harness that runs the primitive twice against THIS repo (the self-project — the inverse of the conductor scripts' self-project *guard*) and asserts the second sweep is idempotent, plus a hermetic unit test that proves the idempotence-assertion logic with injected seams.

### Change 1 — New live-evidence harness `scripts/live-sweep-idempotence-evidence.ts`

Mirror the structure of `scripts/live-conductor-pin-evidence.ts` (evidence-blob + `check()` + `docs/evidence/` artifact + `process.exit`), but **in-process** (the primitive is a direct function call, not an HTTP route — no `route()`/`fetch` layer).

Behavior:
1. **Resolve target = the self/live project.** Default to `process.env.MERMAID_PROJECT ?? process.cwd()` (matches `src/config.ts:90`), overridable via `--project=<path>`.
2. **Positive self-project guard (inverse of the conductor scripts).** Import `isSelfProject` from `../src/services/deploy-service` (`deploy-service.ts:106`). Assert `isSelfProject(project) === true`; **throw** if false — the conductor scripts refuse when the target *is* self (`smoke-conductor-pin-live.ts:247`), this harness refuses when it is *not* self. This is what makes it a "self/live" harness.
3. **Two consecutive sweeps.** Call `runSweepMeasurement(project)` → `s1`, then immediately `runSweepMeasurement(project)` → `s2` (real default `probe`/`runner`/`baseRef`, so GC actually deletes fully-landed branches — the same hygiene the daemon already performs via `runLandedEpicSweep`, `landed-epic-sweep.ts:207`, and what makes the idempotence proof real: run 1's deletions must be visible to run 2).
4. **Idempotence assertions (all POSITIVE, each `check()`-gated):**
   - `check('both sweeps ran against the self-project', s1.project === project && s2.project === project)`
   - `check('second sweep promotes zero missions', s2.promoted.length === 0)` — once s1 promotes queued missions they leave the queue (`promoteQueuedMissions` clears `queuePos`, `mission-store.ts:586`).
   - `check('second sweep deletes zero branches', s2.gcDeleted.length === 0)` — s1's deletes make `revParse` null on run 2 (`landed-epic-sweep.ts:165`).
   - `check('no fully-on-master branches remain after first sweep', s2.fullyOnMasterBranchesRemaining.length === 0)` — post-GC re-probe (`sweep-measurement.ts:96-104`) sees them gone.
   - `check('landedAtDivergence identical across both sweeps', JSON.stringify(s1.landedAtDivergence) === JSON.stringify(s2.landedAtDivergence))` — read-only, stable.
   - `check('gcFlagged identical across both sweeps', JSON.stringify(s1.gcFlagged) === JSON.stringify(s2.gcFlagged))` — ahead>0 branches stay flagged, never deleted (`landed-epic-sweep.ts:163`).
   - `check('queue-starvation set identical across both sweeps', JSON.stringify(s1.sessionsZeroActiveWithQueuedApproved) === JSON.stringify(s2.sessionsZeroActiveWithQueuedApproved))`.
5. **Evidence artifact.** Build a typed `EvidenceBlob { startedAt, finishedAt, project, sweep1: SweepMeasurement, sweep2: SweepMeasurement, assertions, verdict }`, write to `docs/evidence/live-sweep-idempotence-evidence-<finishedAt>.json` via `mkdirSync(...,{recursive:true})` + `writeFileSync` (same as `live-conductor-pin-evidence.ts:281-284`), print JSON to stdout, and `process.exit(verdict === 'PASS' ? 0 : 1)`.

Run line in the header doc comment: `bun run scripts/live-sweep-idempotence-evidence.ts [--project=<path>]`.

### Change 2 — Hermetic unit test `src/services/__tests__/sweep-measurement-idempotence.test.ts`

Gives the primitive its first coverage and mechanically proves the idempotence contract without the real repo, using the `landed-epic-sweep.test.ts` harness pattern (`landed-epic-sweep.test.ts:19-50`: `mkdtempSync` project, `MERMAID_SUPERVISOR_DIR`, injected `GitProbe`, `_closeProject`/`_resetMissionDbCache` teardown).

- **Seed:** one converged-mission epic with `landedAt` set + a `collab/epic/<id8>` branch, and one approved-but-queued mission (via `upsertMission` then `enqueueMission`, `mission-store.ts:531`) whose session has no active mission.
- **Shared stateful seams** (so run 2 observes run 1's mutation, mirroring real git where `makeGitProbe` and `makeBranchGcRunner` read the same repo): `const deleted = new Set<string>()`. The injected `BranchGcRunner.deleteBranch` adds the branch to `deleted`; `revParse` returns null once deleted. The injected `GitProbe` returns `{exists:false,ahead:null,...}` for a branch in `deleted`, else `{exists:true,ahead:0,behind:0,mergeable:true}`.
- **Test `runs twice and the second sweep is idempotent`:** call `runSweepMeasurement(project,{probe,runner})` twice; assert `s1.promoted.length===1 && s2.promoted.length===0`, `s1.gcDeleted.length===1 && s2.gcDeleted.length===0`, `s2.fullyOnMasterBranchesRemaining.length===0`, and `s1.landedAtDivergence`/`s1.gcFlagged` deep-equal their `s2` counterparts.

### Change 3 — Convenience script in `package.json`

Add one line to the `scripts` block: `"evidence:sweep-idempotence": "bun run scripts/live-sweep-idempotence-evidence.ts"` (matches the repo's `bun run scripts/…` convention; not a version field, so it does not violate the `npm version` rule in CLAUDE.md).
