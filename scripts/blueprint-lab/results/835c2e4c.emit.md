## Implementation Blueprint

### Goal
Make a leaf's durable blueprint spec **editable and revisioned** on the `leaf_blueprint` row. Today the durable blueprint row (`src/services/worker-ledger.ts:219-224`, `501-542`) holds only `epicBaseSha` — the actual spec is reattached from the ledger on retry, so editing a rejected leaf's spec is a no-op (the known poisoned-leaf trap). Adding an editable `specJson` with a `specRev` counter gives a durable override slot plus a restore path back to the ledger-derived spec.

### Files to touch
- **`src/services/worker-ledger.ts`** — DDL, additive migration, `LeafBlueprintRow` interface, two new accessors.
- **`src/services/__tests__/worker-ledger.test.ts`** — round-trip / restore / interaction tests.

### Change 1 — DDL columns (`src/services/worker-ledger.ts:219-224`)
Add two columns to the `CREATE TABLE IF NOT EXISTS leaf_blueprint` statement so fresh DBs get them:
```
    epicBaseSha TEXT,
    specJson TEXT,                      -- editable blueprint spec override (null = use ledger-derived)
    specRev INTEGER NOT NULL DEFAULT 0, -- bumps on every edit/restore so consumers can detect change
    recordedAt INTEGER NOT NULL
```

### Change 2 — Additive migration (near `openDb`, after DDL runs, ~`src/services/worker-ledger.ts:179`)
Existing DBs won't get columns from `CREATE TABLE IF NOT EXISTS`. Follow the exact guarded-ALTER idiom used for `leaf_inflight.epoch` (`src/services/worker-ledger.ts:176-179`):
```ts
try {

  if (!lbc.some((c) => c.name === 'specRev')) db.exec('ALTER TABLE leaf_blueprint ADD COLUMN specRev INTEGER NOT NULL DEFAULT 0');
```

### Change 4 — Two new accessors (after `clearLeafBlueprint`, `src/services/worker-ledger.ts:542`)
```ts
/** Edit a leaf's durable blueprint spec override. Upserts the row (so a spec can be set
 *  before any base-SHA record exists) and bumps specRev so consumers detect the change.
 *  Best-effort. */
export function editLeafBlueprintSpec(
  e: { leafId: string; project: string; specJson: string },
  now: number = Date.now(),
): void {
  try {
    openDb().prepare(
      `INSERT INTO leaf_blueprint (leafId, project, epicBaseSha, specJson, specRev, recordedAt)
       VALUES (?,?,NULL,?,1,?)
       ON CONFLICT(leafId) DO UPDATE SET
         specJson=excluded.specJson, specRev=leaf_blueprint.specRev+1, recordedAt=excluded.recordedAt`,
    ).run(e.leafId, e.project, e.specJson, now);
  } catch { /* best-effort */ }
}

/** Restore a leaf's blueprint to the ledger-derived spec by clearing the override.
 *  Bumps specRev so consumers re-read. No-op if the row is absent. Best-effort. */
export function restoreLeafBlueprintSpec(leafId: string, now: number = Date.now()): void {
  try {
    openDb().prepare(
      `UPDATE leaf_blueprint SET specJson=NULL, specRev=specRev+1, recordedAt=? WHERE leafId=?`,
    ).run(now, leafId);
  } catch { /* best-effort */ }
}
```
