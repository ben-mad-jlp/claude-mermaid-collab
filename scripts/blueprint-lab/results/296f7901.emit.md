# Implementation Blueprint

## Problem

`deriveCheapMissionStatus` (`src/services/mission-store.ts:960`) is the facts-free status used by the paginated `/api/supervisor/missions` list and UI badges (the `withFacts:false` path). Its `converged` arm keys off **epics** — `epics.length > 0 && epics.every(e => e.status === 'done')` (line 966) — while the cheap rollup built alongside it keys `converged` off **criteria** — `criteria.length > 0 && capMet === criteria.length` (line 1168). The two disagree in both directions:

- **criteria-converged, zero epics** → `badge = building` while rollup `converged = true` (aad41fd5: 6/6 met, 0/0 epics).
- **all-epics-done, one unmet criterion** → `badge = converged` while rollup `converged = false` (b90bfa21: 6/7).

The fix: key the cheap `converged` off the **capability gauge** (every acceptance criterion met), the same signal the rollup already uses and the same shape as the full `deriveMissionStatus` converged arm (`f.criteria.length > 0 && actions.every(a => a === 'met')`, line 945). `listCriteria` is already read cheaply in `listMissions` (line 1156), so **no facts scan is added** — the criteria just need to be threaded to the two call sites.

## Changes — `src/services/mission-store.ts`

### 1. Re-signature `deriveCheapMissionStatus` (line 960-968)

Replace the `epics` parameter with `criteria` and key `converged` off met criteria:

```ts
export function deriveCheapMissionStatus(
  m: Pick<MissionRow, 'abandonedAt' | 'awaitingApprovalSince'>,
  criteria: readonly { met: boolean }[],
): MissionStatus {
  if (m.abandonedAt != null) return 'abandoned';
  if (m.awaitingApprovalSince != null) return 'unapproved';
  // CAPABILITY gauge, matching the rollup's `converged` flag exactly (and the full
  // deriveMissionStatus converged arm): ≥1 criterion AND all met. Zero criteria is NOT
  // vacuously converged — it reads 'building', same as the facts path's default arm.
  if (criteria.length > 0 && criteria.every((c) => c.met)) return 'converged';
  return 'building';
}
```

Update the JSDoc above it (lines 949-959): the `'converged' arm` is no longer an "all epics done" proxy — it is now the capability gauge (all criteria met), consistent with `converged`. Drop the "not the real capability gauge, which needs criteria verdicts" caveat since criteria verdicts are now used (they're stored columns read via `listCriteria`, still no facts scan).

### 2. Thread criteria to both call sites in `listMissions` (lines 1131-1173)

`listCriteria` is currently read at line 1156, **after** the cheap status is derived at line 1139. Hoist it so both the status derivation and the rollup share the single read. Concretely:

- Move `const criteria = listCriteria(project, node.id);` (line 1156) up to just after `epicsForNode` is computed (before line 1137), keeping the "per-mission indexed lookup, not a facts scan" comment. Remove the old line 1156.
- **Call site A** (line 1138-1139): change `deriveCheapMissionStatus(raw, epicsForNode)` → `deriveCheapMissionStatus(raw, criteria)`.
- **Call site B** (line 1170, inside the cheap `rollup`): change `deriveCheapMissionStatus(mission, epics)` → `deriveCheapMissionStatus(mission, criteria)`. (`epics` is still used for `mechDone`/`mechanical` at lines 1160/1166 — leave those; only the status arg changes.)

No other caller of `deriveCheapMissionStatus` exists (verified: the two sites above are the only references besides the definition and JSDoc).

## Changes — `src/services/__tests__/mission-store.test.ts`

Add `deriveCheapMissionStatus` to the import block (line 12-14). Add a `describe('mission-store: deriveCheapMissionStatus (cheap badge = capability gauge)')` with 5 direct pure-function tests (no DB needed — the function is pure over its two args):

1. **`converged criteria with zero epics reads 'converged' (aad41fd5 direction)`** — `deriveCheapMissionStatus({abandonedAt:null, awaitingApprovalSince:null}, [{met:true},{met:true}])` → `'converged'`. (Epics are irrelevant now; the old code would have returned `'building'`.)
2. **`all criteria not met reads 'building' even if epics would be done (b90bfa21 direction)`** — `deriveCheapMissionStatus({abandonedAt:null, awaitingApprovalSince:null}, [{met:true},{met:false}])` → `'building'`.
3. **`zero criteria is not vacuously converged`** — `deriveCheapMissionStatus({abandonedAt:null, awaitingApprovalSince:null}, [])` → `'building'`.
4. **`abandoned takes precedence over all-criteria-met`** — `deriveCheapMissionStatus({abandonedAt:123, awaitingApprovalSince:null}, [{met:true}])` → `'abandoned'`.
5. **`awaitingApproval takes precedence over all-criteria-met`** — `deriveCheapMissionStatus({abandonedAt:null, awaitingApprovalSince:123}, [{met:true}])` → `'unapproved'`.

These 5 assert both bug directions, zero-criteria non-vacuousness, and terminal precedence, and are stable citable named tests. Threading through the two `listMissions` call sites is covered by tsc (the signature change forces both sites to pass `criteria`) plus the existing `supervisor-missions-bounded.test.ts` zero-fan-out proof (unchanged — `listCriteria` was already on the cheap path).
