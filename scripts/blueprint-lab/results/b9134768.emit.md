## Implementation Blueprint — Deterministic test: same-pass activation + two-run idempotence

### Context
Commit `8e86f868` folded queued-mission promotion into `runLandedEpicSweep` (`src/services/landed-epic-sweep.ts:188-228`): after `reconcileLandedEpics` + `gcEpicBranches`, the composed pass calls `promoteQueuedMissions(project)` (`src/services/mission-store.ts:562-591`) and returns the promoted ids on `RunLandedEpicSweepResult.promoted` (`src/services/landed-epic-sweep.ts:177-181`). No test exercises this composed pass end-to-end. The existing suite (`src/services/__tests__/landed-epic-sweep.test.ts`) only tests `reconcileLandedEpics` and `gcEpicBranches` in isolation, never `runLandedEpicSweep`, and never the promotion fold.

This leaf adds a deterministic, hermetic (injected git probe + runner, no real repo) test covering two properties of the composed pass:
- **same-pass activation**: one `runLandedEpicSweep` call reconciles a landed converged-mission epic AND, in the same pass, promotes a queued sibling mission whose session was freed by that convergence — `result.reconcile.reconciled` contains the epic id and `result.promoted` contains the queued mission id.
- **two-run idempotence**: a second forced `runLandedEpicSweep` reconciles nothing (`reconcile.reconciled` empty) and promotes nothing (`promoted` empty), because the land leaf is already done and the promoted mission is now the session's active non-terminal mission.

### Mechanism these tests pin (already-shipped behavior, read-only confirmed)
- `promoteQueuedMissions` promotes one queued (`active=0`, `queuePos` set), approved (`awaitingApprovalSince==null`), non-terminal, `queuePos`-set candidate per session that has **no active non-terminal mission** (`src/services/mission-store.ts:562-591`). `sessionHasActiveMission` filters on `active && !isMissionTerminal` (`mission-store.ts:633-638`).
- A `converged` mission is terminal (`isMissionTerminal`, `mission-store.ts:42-44`), so it does not block promotion for its session even while its `active` flag lingers.
- A mission with zero criteria derives status `needs-discovery` (`deriveMissionStatus`, `mission-store.ts:946`) → **non-terminal** → a valid promotion candidate.
- `enqueueMission` sets `active=0` + assigns `queuePos` (`mission-store.ts:531-552`).
- `runLandedEpicSweep` needs `force: true` to bypass the 5-min throttle (`shouldRunLandedEpicSweep`, `landed-epic-sweep.ts:23-28,200`).
