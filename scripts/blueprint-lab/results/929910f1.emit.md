## Blueprint: 0700 per-user run dir with port-scoped lock/pid/run files

### Problem
The runtime "run dir" (`port-ownership.ts` `lockDir()`) and its files are neither per-user-isolated nor port-scoped:

- **`lockDir()`** (`src/services/port-ownership.ts:62-65`) resolves to `$XDG_RUNTIME_DIR/mermaid-collab` **or `os.tmpdir()/mermaid-collab`**. On Linux without XDG (systemd `ExecStartPre` with a stripped env, headless), `os.tmpdir()` is the world-writable, world-readable **shared `/tmp`** — every user's lockfile lands in one `/tmp/mermaid-collab` dir, created with default (umask-masked, typically `0755`) perms. A foreign user can read another user's lock (pid/exePath disclosure) or plant files.
- **Lock file names are not port-scoped**: `server.lock` (`:69`) and `server.takeover.lock` (`:74`) are fixed strings. Two collab servers on different ports (dev `:9002` + a second `:9003`, or the desktop app + a bare CLI on an override port) collide on the same lockfile — one overwrites the other's ownership record, defeating the handshake.
- **The CLI pid file** (`bin/mermaid-collab.ts:27`, `server.pid`) has the same fixed-name collision problem across ports.

### Fix
Give `port-ownership.ts` a **per-user, `0700` run dir** and **port-scoped** lock / takeover-mutex / pid file names; thread the resolved port through the lock lifecycle; move the CLI pid file into that run dir.

---

### 1. `src/services/port-ownership.ts` — per-user 0700 run dir + port-scoped files

**a. Add a uid helper and an internal 0700-dir ensurer** (near `serverOwner`, ~`:107`):

```ts
/** POSIX uid of this process, or 0 where getuid is unavailable (Windows/legacy). */
export function runtimeUid(): number {
  return typeof process.getuid === 'function' ? process.getuid() : 0;
}

/** Create `dir` (recursive) and force it to 0700. mkdir's mode arg is masked by
 *  umask and a pre-existing dir keeps its old perms, so chmod explicitly. */
function ensureDir700(dir: string): void {
  fs.mkdirSync(dir, { recursive: true, mode: 0o700 });
  try { fs.chmodSync(dir, 0o700); } catch { /* best-effort on platforms w/o chmod */ }
}
```

**b. Make `lockDir` per-user** (`:62-65`) — suffix the dir with the uid so the shared-`/tmp` fallback (and XDG, harmlessly) is isolated:

```ts
export function lockDir(env: NodeJS.ProcessEnv = process.env): string {
  const base = env.XDG_RUNTIME_DIR && env.XDG_RUNTIME_DIR.trim() ? env.XDG_RUNTIME_DIR : os.tmpdir();
  return path.join(base, `mermaid-collab-${runtimeUid()}`);
}

/** Ensure the per-user run dir exists at 0700. Callable by starters before writing. */
export function ensureRunDir(env: NodeJS.ProcessEnv = process.env): string {
  const dir = lockDir(env);
  ensureDir700(dir);
  return dir;
}
```

**c. Port-scope the file paths** (`:68-75`) — add a `port` param, defaulting to `getConfiguredPort()` (already imported, `:17`), and add a `pidPath`:

```ts
export function lockPath(env: NodeJS.ProcessEnv = process.env, port: number = getConfiguredPort()): string {
  return path.join(lockDir(env), `server-${port}.lock`);
}
export function takeoverMutexPath(env: NodeJS.ProcessEnv = process.env, port: number = getConfiguredPort()): string {
  return path.join(lockDir(env), `server-${port}.takeover.lock`);
}
/** Port-scoped pid file inside the per-user run dir (the CLI's start/stop/status record). */
export function pidPath(env: NodeJS.ProcessEnv = process.env, port: number = getConfiguredPort()): string {
  return path.join(lockDir(env), `server-${port}.pid`);
}
```

**d. Ensure 0700 wherever run-dir files are created.** In `acquireExclusive` (`:118`) replace `fs.mkdirSync(path.dirname(filePath), { recursive: true });` with `ensureDir700(path.dirname(filePath));`. In `writeLock` (`:166`) replace its `fs.mkdirSync(path.dirname(p), { recursive: true });` with `ensureDir700(path.dirname(p));`.

**e. Thread `port` through the lock lifecycle:**
- `writeLock(data)` (`:164-168`): derive path from the record's own port — `const p = lockPath(env, data.port);`.
- `readLock(env, port = getConfiguredPort())` (`:171`): read `lockPath(env, port)`.
- `releaseLock(pid, env, port = getConfiguredPort())` (`:183`): unlink `lockPath(env, port)`.
- `acquireTakeoverMutex(env, contents, ttlMs, port = getConfiguredPort())` (`:144`): use `takeoverMutexPath(env, port)`.
- In `performHandshake` (`:314`): after `const port = deps.port ?? lock?.port ?? getConfiguredPort();` (`:332`), the `lock` at `:331` must be read with that port. Since `port` depends on `lock?.port`, first read with the default, then re-resolve: keep `const lock = readLock(env);` (default port), compute `port`, and pass `port` explicitly to every subsequent `lockPath(env, port)` (`:341,350,351`), `takeoverMutexPath(env, port)` (`:374,427`), and `acquireTakeoverMutex(env, String(process.pid), undefined, port)` (`:370,409`).

### 2. `bin/mermaid-collab.ts` — port-scoped pid file in the 0700 run dir

- Extend the import (`:17-22`) to add `ensureRunDir, pidPath`.
- Replace `const PID_FILE = join(DATA_DIR, 'server.pid');` (`:27`) with a resolved constant: `const PID_FILE = pidPath(process.env, Number(REQUEST_PORT));` (place it after `REQUEST_PORT` at `:32`). `REQUEST_PORT` is `process.env.PORT || 9002` (`:32`), matching the port the server is told to bind and the handshake resolves — so start/stop/status share one path.
- In `ensureDataDir` (`:34-36`), also create the run dir at 0700: add `ensureRunDir(process.env);` so the pid write target exists with correct perms. `LOG_FILE` stays in `DATA_DIR` (home, already per-user).

### 3. `src/server.ts` — release the port-scoped lock

The server writes the lock keyed by `actualPort` (`:729-735`, `data.port`), so its three `releaseLock()` calls (`:763,779,799`) must target the same port. Change each to `releaseLock(process.pid, process.env, actualPort);` (`actualPort` is in scope from `:695`).

### 4. `src/services/__tests__/port-ownership-rundir.test.ts` — new test (create)

`bun:test`, injected `env` with `XDG_RUNTIME_DIR` pointed at a `mkdtempSync` scratch dir (mirror `port-ownership-uid.test.ts:19-25`). Cases:
- **`run dir is created 0700`** — `ensureRunDir(env)`; assert `(fs.statSync(lockDir(env)).mode & 0o777) === 0o700`.
- **`lock path is port-scoped`** — `lockPath(env, 9003)` and `takeoverMutexPath(env, 9003)` end with `server-9003.lock` / `server-9003.takeover.lock`; `lockPath(env, 9002) !== lockPath(env, 9003)`.
- **`pid path is port-scoped and inside the run dir`** — `pidPath(env, 9003)` ends with `server-9003.pid` and `path.dirname(pidPath(env, 9003)) === lockDir(env)`.
- **`run dir is per-user (uid in path)`** — `lockDir(env)` contains `mermaid-collab-${runtimeUid()}`.
- **`writeLock → readLock round-trips per port`** — `writeLock({pid,exePath,version,port:9003,owner}, env)`; `readLock(env, 9003)` returns it while `readLock(env, 9002)` returns `null`.

3. `lockPath()`, `takeoverMutexPath()`, and new `pidPath()` in `src/services/port-ownership.ts` accept a `port` param and produce `server-<port>.{lock,takeover.lock,pid}` names.

5. `src/server.ts` passes `actualPort` to all three `releaseLock(...)` calls.
