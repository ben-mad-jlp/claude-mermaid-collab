## Blueprint: Fix `findLandedAtDivergence` — landed satisfies, stranded still violates

### Problem
`findLandedAtDivergence` (`src/services/invariant-check.ts:152`) currently enforces a **biconditional** equivalence at line 165:

```ts
if (hasDoneLand !== (t.landedAt != null)) { ... }
```

This flags **both** directions as divergence:
1. `hasDoneLand && landedAt == null` — a done `[LAND]` leaf exists but the epic was never stamped. This is a genuine **stranded** epic (landed-by-leaf, property missing) → **must still violate**.
2. `!hasDoneLand && landedAt != null` — the epic has `landedAt` set but no done land leaf.

Direction (2) is now a **false positive**. Per the landing model (`landedAt` is the epic property that *replaces* the land leaf across all land paths — mission `48e1a624`), an epic can be legitimately landed with `landedAt` stamped and **no land leaf at all**. Landing-via-property must **satisfy** the invariant, not trip it.

### Fix
Change the biconditional to a one-directional implication: *a done land-leaf child implies `landedAt` is stamped*. Only the stranded case (direction 1) remains a violation.
