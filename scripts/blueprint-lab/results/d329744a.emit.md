### Problem (grounded)
`scripts/blueprint-lab/corpus.ts:130-169` `buildCorpus()` greedily keeps **every** eligible commit in log order and only stops once `cases.length >= MIN_CASES && seenKinds.size >= MIN_LEAF_KINDS` (line 165). There is currently **no test** for the corpus (empty `__tests__`), so the invariant is unenforced.

### Change shape

**1. `scripts/blueprint-lab/corpus.ts` — replace greedy accumulation with kind-balanced round-robin selection.**

- Keep `MIN_CASES = 10` / `MIN_LEAF_KINDS = 3` (lines 39-40) but **export** them (they become the test's invariants).
- Add two exported constants after them:
  - `export const TARGET_CASES = 12;` — upper bound so the corpus is right-sized (≥ `MIN_CASES`, far below today's 54).
  - `export const MAX_PER_KIND = 6;` — no single leafKind may dominate.
- Add an exported `interface EligibleCommit { commit: RawCommit; touchedFiles: string[]; leafKind: DiffLeafKind; }`.
- Extract the eligibility+classify pass (currently lines 136-145) into `function collectEligible(commits: RawCommit[]): EligibleCommit[]` — applies the same three filters (single-parent line 137, `collab:`/`Merge:` line 138, duplicate-subject line 139), calls `getTouchedFiles` + `classifyLeafKind`, and pushes an `EligibleCommit`. This does **not** call the expensive `extractChangedSymbols`.
- Add `export function selectBalanced(eligible: EligibleCommit[], target: number): EligibleCommit[]`: bucket `eligible` by `leafKind` (log order preserved within bucket); order kinds rarest-bucket-first; round-robin one-per-kind per round, skipping a kind once it hits `MAX_PER_KIND` or its bucket is exhausted, stopping at `target` or when a full round makes no progress; return `eligible.filter(e => taken.has(e))` so the result stays in recency (log) order. This guarantees every present kind gets representation before any common kind floods.
- Add `function assembleCase(e: EligibleCommit): CorpusCase` holding the existing case-construction (lines 148-162) plus the now-selected-only `extractChangedSymbols(e.commit.sha, e.commit.parents[0], e.touchedFiles)` call — so the expensive `-U0` diff runs for ≤12 commits, not 54.
- Rewrite `buildCorpus()` (lines 130-169) to: `const eligible = collectEligible(parseLog()); return selectBalanced(eligible, TARGET_CASES).map(assembleCase);` — dropping the `seenKinds`/`previousKeptSubject`-in-loop/break machinery (the dup-subject filter moves into `collectEligible`).

With `feature135/fix24/test7` and `MAX_PER_KIND=6`, round-robin to 12 yields ~4/4/4 across all 3 kinds — ≥10 cases, ≤12, 3 kinds, balanced.
