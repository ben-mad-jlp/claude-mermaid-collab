### Change shape

**1. `src/services/worker-ledger.ts` — add `seedLeafBlueprintSpec` (new export, placed right after `restoreEditableBlueprint`, ~line 580):**

```ts
/** Seed the durable editable contract (leaf_blueprint.specJson) from a freshly-authored
 *  blueprint's raw output text. Parses the trailing v2 json fence into a DiffContract and
 *  stores its NORMALIZED render at specRev 0 — the canonical baseline the edit accessors
 *  bump from and restoreEditableBlueprint splices. epicBaseSha is passed through so the
 *  upsert never nulls the durable base SHA. No parseable contract ⇒ no-op (the base-SHA
 *  persist already recorded the row). Best-effort; never throws. */
export function seedLeafBlueprintSpec(
  e: { leafId: string; project: string; epicBaseSha?: string | null },
  blueprintText: string,
  now: number = Date.now(),
): void {
  try {
    const contract = parseDiffContract(blueprintText);
    if (!contract) return;
    recordLeafBlueprint(
      { leafId: e.leafId, project: e.project, epicBaseSha: e.epicBaseSha ?? null,
        specJson: renderContract(contract), specRev: 0 },
      now,
    );
  } catch { /* best-effort — never break the run */ }
}
```

Note: `epicBaseSha` is forwarded because `recordLeafBlueprint`'s upsert sets `epicBaseSha=excluded.epicBaseSha` unconditionally (worker-ledger.ts:536) — omitting it would null the durable base. Non-parseable text is a no-op, so small-tier/synthetic blueprints leave the existing base-SHA row untouched.
