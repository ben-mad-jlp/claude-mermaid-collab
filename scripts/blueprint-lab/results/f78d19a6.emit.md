## Blueprint: add `^scripts/` test lane

### Root cause (verified)
`.collab/project.json` (`.collab/project.json:8-11`) declares exactly two `gate.tests` lanes: `^src/` → `bun test {file}` and `^ui/` → `bunx vitest`. When a leaf's change-set includes `scripts/blueprint-lab/__tests__/harness.test.ts`, `routeSpecsToLanes` (`src/services/leaf-gate.ts:436-455`) finds no matching lane (matches are anchored with `^`), so `runLeafGate` returns `status:'error'` via the "config gap" arm (`src/services/leaf-gate.ts:302-317`) — the gate can't verify the leaf.

`SPEC_FILE_RE` (`src/services/gate-runner.ts:361`) already accepts `.test.ts`, so a `scripts/…/*.test.ts` path IS treated as a spec; it just has nowhere to route. The existing `scripts/__tests__/parent-epic-under-mission.test.ts` imports from `bun:test`, so the correct runner for `scripts/` specs is `bun test {file}` (identical to the `^src/` lane).

### Change shape

**1. `.collab/project.json` — add a third lane** (`.collab/project.json:8-11`)

Insert a `^scripts/` lane into the `gate.tests` array. Matches are disjoint from `^src/`/`^ui/`, so placement is free; put it after `^src/`:

```json
"tests": [
  { "match": "^src/", "command": "bun test {file}" },
  { "match": "^scripts/", "command": "bun test {file}" },
  { "match": "^ui/", "command": "bunx vitest --run {files}", "cwd": "ui" }
]
```

This produces a `per-file` lane (contains `{file}`), consistent with `normalizeLanes` (`src/services/leaf-gate.ts:113-122`). No `cwd` — `scripts/` specs resolve from the repo root, matching how `scripts/__tests__/…` imports (`../parent-epic-under-mission.ts`).

**2. `src/services/__tests__/leaf-gate.test.ts` — update the real-config guard** (`src/services/__tests__/leaf-gate.test.ts:469-482`)

That test hard-asserts `manifest.gate!.tests!.length).toBe(2)` (line 474) and only lanes `[0]`/`[1]` — adding a lane breaks it, so it must be updated in the same leaf. Rename it to reflect three lanes and assert the new `^scripts/` lane:

- Change `expect(...tests!.length).toBe(2)` → `toBe(3)` (line 474).
- Keep `tests![0].match === '^src/'` (line 475).
- Add `expect(manifest.gate!.tests!.some((l) => l.match === '^scripts/')).toBe(true)`.
- Add `expect(manifest.gate!.tests!.find((l) => l.match === '^scripts/')!.command).toContain('bun test')`.
- Keep the `^ui/`/vitest assertions, indexing the ui lane by find (`tests!.find((l) => l.match === '^ui/')`) rather than fixed index `[1]`, since ui is now index 2.
- Rename the `it(...)` title from `'real .collab/project.json of THIS repo has the dual-lane config'` to `'real .collab/project.json of THIS repo has src/scripts/ui test lanes'`.

### Verification
`bun test src/services/__tests__/leaf-gate.test.ts` passes; the updated real-config test proves the `^scripts/` lane is present in the actual manifest. A leaf whose change-set is `scripts/blueprint-lab/__tests__/harness.test.ts` now routes to the new lane instead of landing in `unmatched`.
