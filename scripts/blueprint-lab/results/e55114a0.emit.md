## Blueprint: Honor the pin in `runConductorPass` with no-fallback and lazy auto-clear

### Context
`runConductorPass` (`src/services/conductor-pass.ts:91-143`) currently selects the mission to drive by iterating **all active missions** and taking the first that qualifies (`src/services/conductor-pass.ts:96-105`). The explicit pin — `conductorTargetMissionId`, with accessors `getConductorTargetMission` / `setConductorTargetMission` (`src/services/supervisor-store.ts:483-493`) — is persisted but **not yet honored**. This leaf wires the pin into selection with two behaviors:

- **No-fallback**: when the pin names a mission, the pass drives *only* that mission. If the pinned mission is live but not actionable right now (e.g. awaiting approval), the pass waits — it never silently drives a different active mission.
- **Lazy auto-clear**: when the pinned mission is terminal (`isMissionTerminal`, `src/services/mission-store.ts:41-43` — converged/abandoned) or no longer exists, the pass drops the stale pin via `setConductorTargetMission(project, null)` and reverts to first-active selection *this same pass*.

2. **`ConductorPassResult.reason` union** (line 83): add a new member `'pin-not-actionable'` for the honored-but-waiting case.

3. **Selection block** (replace lines 96-105). Extract a `qualifies(row)` predicate mirroring today's condition (`row.awaitingApprovalSince == null && row.status != null && !['unapproved','abandoned','converged'].includes(row.status)`), then:
   - Read `const pinnedId = getConductorTargetMission(project)`.
   - If `pinnedId` is set: look up `const row = getMission(project, pinnedId)` and `const summary = listMissions(project).find((m) => m.node.id === pinnedId)`.
     - If `!row || !summary || isMissionTerminal(row)` → **lazy auto-clear**: `setConductorTargetMission(project, null)` and leave `target` unset (falls through to first-active loop below).
     - Else if `qualifies(row)` → `target = { summary, row }` (drive the pin).
     - Else → **no-fallback**: `return { ran: false, reason: 'pin-not-actionable', missionId: pinnedId }`.
   - If `target` still unset, run the existing first-active loop (`listMissions(project).filter((m) => m.mission.active)` → first `qualifies(row)`).
   - Keep the existing `if (!target) return { ran: false, reason: 'no-actionable-mission' }` guard (line 105).

   Everything downstream (lines 106-143) is unchanged — it already reads `target.row.todoId` / `target.summary`.
