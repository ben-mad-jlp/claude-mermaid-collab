## Blueprint — Corpus builder: mine ≥10 historical landed leaves across ≥3 leafKinds with real base..HEAD diffs

### Context
The epic is building the **v2 diff-contract** (`src/services/diff-contract.ts`: `DiffLeafKind`, `CONTRACT_STRICTNESS_MATRIX`, `validateContractForKind`, `parseDiffContract`). That module is "NOT wired into any pipeline yet" and needs a **calibration corpus** of real landed leaves to test the leafKind classification/strictness against actual diffs. This leaf produces that corpus + the builder that mines it.

Key facts established by inspection:
- Landed leaves are conventional-commit-prefixed commits on `master` (`git log master --pretty=%s`): **feat** (~1300), **fix** (~470), **refactor** (~40), **test** (~30), **chore** (~55). All 5 `DiffLeafKind`s are reachable; ≥3 is trivially satisfiable.
- Epic-level merges (`collab: land epic … → master`, `Merge …`) and version bumps (`6.18.2`) are **not** leaves and must be excluded.
- A squashed landed leaf's `base..HEAD` diff == its parent..commit patch (`git show <sha>`), since the commit is the whole leaf.
- Git-runner convention in this repo: `execFileSync('git', ['-C', cwd, …], { encoding: 'utf8' })` (see `src/services/leaf-commit-scope.ts:1,49`). Injecting the runner keeps the miner unit-testable.
- Test runner for this module family is **bun:test** (`src/services/__tests__/diff-contract.test.ts:1`). Fixture-corpus precedent: `src/services/__fixtures__/todo-kind-cases.json`.

### Files to create

**1. `src/services/diff-corpus.ts`** — the builder + classifier.

Import `type DiffLeafKind` and `DIFF_LEAF_KINDS` from `./diff-contract`.

Public surface:
- `export type GitRunner = (args: string[]) => string;` — invoked as `run(['log', …])`; default = `execFileSync('git', ['-C', cwd, ...args], { encoding: 'utf8', maxBuffer })`.
- `export interface CorpusLeaf { sha: string; subject: string; leafKind: DiffLeafKind; baseSha: string; changedFiles: string[]; additions: number; deletions: number; diff: string; }`
- `export const CORPUS_MIN_ENTRIES = 10;` and `export const CORPUS_MIN_KINDS = 3;`
- `export function classifyLeafKind(subject: string): DiffLeafKind | null` — regex `^(\w+)(\([^)]*\))?!?:\s` to grab the conventional prefix; map `feat→'feature'`, `fix→'fix'`, `refactor→'refactor'`, `test→'test'`, `chore|build|ci|infra→'infra'`; **return `null`** for anything else (`collab`, `Merge …`, `revert`, `docs`, `style`, bare version `6.18.2`). Validate the mapped value against `DIFF_LEAF_KINDS` before returning.
- `export interface MineOptions { cwd?: string; maxCommits?: number; run?: GitRunner; minEntries?: number; minKinds?: number; }`
- `export function mineLandedLeafCorpus(opts?: MineOptions): CorpusLeaf[]`:
  1. `run(['log', '--no-merges', `-n${maxCommits ?? 4000}`, '--format=%H%x1f%s'])`, split on `\x1e`/newlines, parse `sha \x1f subject`.
  2. For each, `classifyLeafKind(subject)`; skip `null`.
  3. Dedup by `subject` (keep first/newest occurrence — landed leaves recur once per forward-integration).
  4. For each survivor: `baseSha = run(['rev-parse', `${sha}^`]).trim()`; `changedFiles`/`additions`/`deletions` from `run(['show', '--numstat', '--format=', sha])`; `diff = run(['show', '--format=', sha])` (the real base..HEAD patch), truncated to a `MAX_DIFF_CHARS` cap (non-empty preserved).
  5. Accumulate until `minEntries` (default `CORPUS_MIN_ENTRIES`) entries **and** `minKinds` (default `CORPUS_MIN_KINDS`) distinct leafKinds are collected; then continue to end-of-window but return the full set.
  6. If the window yields `< minEntries` entries or `< minKinds` distinct kinds, **throw** `Error('diff-corpus: mined N leaves across K kinds, need ≥10/≥3')` so materialization fails loudly.
- `export interface CorpusFixture { schemaVersion: 1; generatedFromRef: string; minEntries: number; minKinds: number; leaves: CorpusLeaf[]; }`
- `export function buildCorpusFixture(opts?: MineOptions): CorpusFixture` — wraps `mineLandedLeafCorpus`, sets `generatedFromRef = run(['rev-parse','HEAD']).trim()`.
- CLI tail: `if (import.meta.main) { process.stdout.write(JSON.stringify(buildCorpusFixture(), null, 2) + '\n'); }` — so the fixture is materialized deterministically via `bun run src/services/diff-corpus.ts > src/services/__fixtures__/diff-corpus.json`.

**2. `src/services/__fixtures__/diff-corpus.json`** — the materialized corpus artifact, produced by running the CLI tail above against this repo's real history. `CorpusFixture` shape with `leaves.length ≥ 10`, ≥3 distinct `leafKind` values, every leaf carrying a non-empty `diff` and distinct `sha`/`baseSha`. (Implement node runs the CLI once and checks in the output.)

**3. `src/services/__tests__/diff-corpus.test.ts`** (bun:test):
- `classifyLeafKind maps conventional prefixes to leafKinds` — asserts `feat(ui): …`→`'feature'`, `fix: …`→`'fix'`, `refactor(mcp): …`→`'refactor'`, `test: …`→`'test'`, `chore(cleanup): …`→`'infra'`, and `null` for `collab: land epic … → master`, `Merge branch 'master'`, `6.18.2`.
- `mineLandedLeafCorpus returns ≥10 entries across ≥3 leafKinds with non-empty diffs` — inject a fake `GitRunner` returning a synthetic 12-commit log spanning feat/fix/test/refactor plus numstat/show stubs (deterministic, no real git); assert `length ≥ CORPUS_MIN_ENTRIES`, `new Set(map leafKind).size ≥ CORPUS_MIN_KINDS`, every `diff` non-empty, `baseSha !== sha`.
- `checked-in diff-corpus fixture satisfies corpus invariants` — `import fixture from '../__fixtures__/diff-corpus.json'`; assert `fixture.leaves.length >= 10`, distinct leafKinds `>= 3`, every leaf `diff.length > 0` and `sha`/`baseSha` present.

### Acceptance criteria (positive, citable)
1. `src/services/diff-corpus.ts` exports `classifyLeafKind` mapping each conventional-commit prefix to a `DiffLeafKind` (or `null`) — citable at its `export function classifyLeafKind` line.
2. `src/services/diff-corpus.ts` exports `mineLandedLeafCorpus` that walks `git log --no-merges`, dedups landed leaves, and attaches each leaf's real `base..HEAD` diff via `git show` — citable at its `export function mineLandedLeafCorpus` line.
3. `src/services/diff-corpus.ts` exports `CorpusLeaf` carrying `{ sha, leafKind, baseSha, diff, changedFiles, additions, deletions }` — citable at its `export interface CorpusLeaf`.
4. `src/services/__fixtures__/diff-corpus.json` exists with `leaves.length ≥ 10` spanning ≥3 distinct `leafKind` values, each leaf carrying a non-empty `diff`.
5. `src/services/__tests__/diff-corpus.test.ts` contains the test `checked-in diff-corpus fixture satisfies corpus invariants` asserting the ≥10-entry / ≥3-kind / non-empty-diff invariants.

### Out of scope
- Wiring the corpus into `validateContractForKind`/any pipeline (a later leaf consumes it).
- Inferring a leaf's `DiffLeafKind` from diff *content* (classification here is by commit-message prefix only).
