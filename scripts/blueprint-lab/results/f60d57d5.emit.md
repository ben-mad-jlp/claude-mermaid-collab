## Blueprint: Blueprint-lab contract scorer

### Context
- `src/services/diff-contract.ts` already ships the pure primitives: `validateContractForKind(contract, leafKind)` (`diff-contract.ts:99`) returns `{underspecified:true, missingField}` or `{underspecified:false}` per the `CONTRACT_STRICTNESS_MATRIX` (`diff-contract.ts:69`); plus `DiffContract`, `DiffLeafKind`, `DiffRequirementKind`, `DIFF_LEAF_KINDS`.
- `scripts/blueprint-lab/emit.ts` runs the real blueprint node per corpus case and writes `results/run.json` as `{total, parsed, unparsed, results: EmitResult[]}` where `EmitResult = {id, leafKindExpected, contract: DiffContract|null, rawText}` (`emit.ts:106`, `:149`).
- `scripts/blueprint-lab/corpus.ts` exports `CORPUS: CorpusCase[]` where each case carries the **actual** landed diff: `leafKind`, `diff.touchedFiles`, `diff.changedSymbols` (`corpus.ts:17`, `:148`).
- The backend test runner (`scripts/test-backend.ts:11,21,44`) only scans `*.test.ts` **under `src/`**. Tests placed under `scripts/` are NOT run by the suite. Therefore the pure, testable scoring logic must live in `src/services/` (mirroring how `diff-contract.ts` is pure logic in `src/` while `emit.ts`/`corpus.ts` are the lab harness in `scripts/`). `src/` must not import from `scripts/`, so the scorer defines a local minimal `ActualDiff` shape rather than importing `CorpusCase`.

### Change shape

**1. New pure module `src/services/blueprint-scorer.ts`** (standalone leaf module, no daemon wiring — mirrors `diff-contract.ts`). Imports only from `./diff-contract`. Exports:

- `interface ActualDiff { leafKind: DiffLeafKind; touchedFiles: string[]; changedSymbols: string[] }` — the actual landed-diff facts, decoupled from `scripts/blueprint-lab/corpus.ts`'s `CorpusCase`.
- `interface ValidationOutcome { accepted: boolean; missingField: DiffRequirementKind | null }`.
- `function scoreValidation(contract: DiffContract, leafKind?: DiffLeafKind): ValidationOutcome` — calls `validateContractForKind(contract, leafKind ?? contract.leafKind)`; maps `{underspecified:false}` → `{accepted:true, missingField:null}` and `{underspecified:true, missingField}` → `{accepted:false, missingField}`. Enables scoring the contract either against its own declared `leafKind` or against the corpus-actual `leafKind`.
- `interface MatchScore { leafKindMatch: boolean; fileRecall: number; filePrecision: number; symbolRecall: number; symbolPrecision: number }`.
- `function declaredVsActual(contract: DiffContract, actual: ActualDiff): MatchScore` — `leafKindMatch = contract.leafKind === actual.leafKind`; declared files = `[...contract.filesToCreate, ...contract.filesToEdit]` (deduped Set); declared symbols = symbols from `symbol-present` requirements (`requirements.filter(r=>r.kind==='symbol-present').map(r=>r.symbol)`, deduped). Recall = `|declared ∩ actual| / |actual|`, precision = `|declared ∩ actual| / |declared|`. Guard division-by-zero: when a denominator is 0, that rate is `1` if the other set is also empty, else `0` (documented in a comment).
- `interface RunScorecard { total; parsed; unparsed; validation: { accepted; rejected; acceptRate; byLeafKind: Record<DiffLeafKind,{accepted;rejected}>; missingFieldHistogram: Record<DiffRequirementKind,number> }; match: { leafKindMatchRate; meanFileRecall; meanFilePrecision; meanSymbolRecall; meanSymbolPrecision }; perCase: Array<{id; parsed; validation: ValidationOutcome|null; match: MatchScore|null}> }`.
- `function scoreEmitRun(entries: Array<{ id: string; contract: DiffContract | null; actual: ActualDiff }>): RunScorecard` — for each entry: null contract → counted in `unparsed`, `perCase` entry with `parsed:false` and null validation/match; parsed → `scoreValidation(contract, actual.leafKind)` (score against the corpus-actual leafKind, the ground truth) and `declaredVsActual`. Aggregates: acceptance/rejection enumeration (counts + `byLeafKind` keyed by the actual leafKind + `missingFieldHistogram` counting each rejection's `missingField`), and the declared-vs-actual match rates as means over parsed entries. Histogram initialized with all `DIFF_LEAF_KINDS` / all three `DiffRequirementKind` keys at 0 so the shape is stable.

**2. New lab runner `scripts/blueprint-lab/score.ts`** (thin I/O, mirrors `emit.ts`/`reviewer-lab/run.ts` shape). Reads `results/run.json`, joins each `EmitResult.id` to its `CORPUS` case to build `ActualDiff` (`{leafKind: case.leafKind, touchedFiles: case.diff.touchedFiles, changedSymbols: case.diff.changedSymbols}`), calls `scoreEmitRun`, writes `scripts/blueprint-lab/results/score.json`, and prints a scorecard (accept rate, per-leafKind accept/reject, missingField histogram, leafKind-match rate + mean file/symbol recall/precision). Imports `scoreEmitRun` from `../../src/services/blueprint-scorer` and `CORPUS` from `./corpus`.

**3. New test `src/services/__tests__/blueprint-scorer.test.ts`** (`bun:test`, mirrors `diff-contract.test.ts:1`). Named tests:
- `scoreValidation accepts a fully-specified feature contract` — feature contract with symbol-present + named-test → `{accepted:true, missingField:null}`.
- `scoreValidation rejects a feature contract missing named-test with missingField` — feature contract with only symbol-present → `{accepted:false, missingField:'named-test'}`.
- `declaredVsActual computes leafKind match and file/symbol recall and precision` — asserts exact recall/precision fractions on a hand-built contract+ActualDiff (including a partial overlap).
- `scoreEmitRun enumerates acceptance/rejection and mean match rates` — mixed batch (one accepted, one rejected, one null) → asserts `validation.accepted/rejected`, `missingFieldHistogram`, `unparsed`, and a `match` mean.

### Acceptance criteria (positive, citable)
1. `src/services/blueprint-scorer.ts` exports `scoreValidation` that delegates to `validateContractForKind` and returns `{accepted, missingField}`.
2. `src/services/blueprint-scorer.ts` exports `declaredVsActual` returning `{leafKindMatch, fileRecall, filePrecision, symbolRecall, symbolPrecision}` against an `ActualDiff`.
3. `src/services/blueprint-scorer.ts` exports `scoreEmitRun` producing a `RunScorecard` with the validation accept/reject enumeration (`byLeafKind` + `missingFieldHistogram`) and the mean declared-vs-actual match rates.
4. `scripts/blueprint-lab/score.ts` reads `results/run.json` + `CORPUS`, calls `scoreEmitRun`, and writes `results/score.json`.
5. `src/services/__tests__/blueprint-scorer.test.ts` contains the four named tests above and passes under `bun run scripts/test-backend.ts blueprint-scorer`.

### Out of scope
- Wiring the scorer into the daemon pipeline or any gate.
- Modifying `diff-contract.ts`, `emit.ts`, or `corpus.ts`.
- Re-running the live emission (`emit.ts`) to regenerate `run.json`.
