The blueprint-lab has three modules — `corpus.ts` (mines historical leaves), `emit.ts` (spawns the real blueprint node → `results/run.json`), and `score.ts` (scores contracts vs actual diffs → `results/score.json`). ### Change shape

**1. CREATE `scripts/blueprint-lab/lab.ts`** — the orchestrator + report + gate. Structure:

- Module docstring: "blueprint-lab orchestrator — chains emit.ts → score.ts, renders results/report.md, and emits a PASS/FAIL GATE verdict (exit 0/1) against declared thresholds. Reuse existing run.json/score.json via SKIP_EMIT=1."
- Imports: `readFileSync, writeFileSync` from `node:fs`; `spawnSync` from `node:child_process`; `join` from `node:path`. Reuse `import.meta.dir` for the `results/` dir like emit/score do (`emit.ts:22`).
- Types mirrored from score.ts so the report is typed: a local `interface Aggregate` matching `AggregateStats` (`score.ts:102`) and `interface ScoreFile { scores: any[]; aggregate: Aggregate }`, plus `interface RunFile { total: number; parsed: number; unparsed: string[]; results: any[] }`.
- **`const GATE`** — the declared thresholds object with env overrides and literal defaults:
  ```
  minParseRate (BP_MIN_PARSE_RATE, default 0.9)
  minMeanMatchRate (BP_MIN_MATCH_RATE, default 0.4)
  minAcceptRate (BP_MIN_ACCEPT_RATE, default 0.7)   // validationCounts.accept / total
  maxKindMismatchRate (BP_MAX_KIND_MISMATCH, default 0.3)  // leafKindMismatchCount / total
  ```
- **`interface GateCheck`** = `{ name: string; actual: number; comparison: 'gte' | 'lte'; threshold: number; ok: boolean }`.
- **`function evaluateGate(run: RunFile, score: ScoreFile): { pass: boolean; checks: GateCheck[] }`** — computes the four rates from `run`/`score.aggregate` (parseRate = `run.parsed / run.total`; acceptRate = `(agg.validationCounts.accept ?? 0) / agg.total`; kindMismatchRate = `agg.leafKindMismatchCount / agg.total`; meanMatchRate = `agg.meanMatchRate`), builds one `GateCheck` per threshold (`gte` for the three minimums, `lte` for the max), and `pass = checks.every(c => c.ok)`. Guard `total === 0` → all rates `0` (division-safe, mirrors `score.ts:74` `actualCount > 0` guard).
- **`function renderReport(run: RunFile, score: ScoreFile, gate: { pass; checks }): string`** — returns a markdown string: a `# blueprint-lab report` header, a **GATE** section with a `| check | actual | cmp | threshold | ok |` table (one row per `gate.checks`) and a bold `**GATE: PASS/FAIL**` line, an **Aggregate** section (parsed/total, meanMatchRate %, leafKind mismatches, `validationCounts`), and a **Per-case** table with columns `id | leafKind exp/act | validation | matchRate` driven by `score.scores`.
- **`async function main()`**:
  1. `const only = process.argv.slice(2)`.
  2. Unless `process.env.SKIP_EMIT === '1'`: `spawnSync('bun', [join(import.meta.dir,'emit.ts'), ...only], { stdio: 'inherit' })`; if `status !== 0` print error and `process.exit(2)`.
  3. `spawnSync('bun', [join(import.meta.dir,'score.ts')], { stdio: 'inherit' })`; on non-zero → `process.exit(2)`.
  4. Read `results/run.json` and `results/score.json` via `readFileSync`+`JSON.parse`.
  5. `const gate = evaluateGate(run, score)`.
  6. `writeFileSync(join(OUT,'report.md'), renderReport(run, score, gate))`.
  7. `console.log` each check line + `GATE: PASS|FAIL`.
  8. `process.exit(gate.pass ? 0 : 1)` — the CI-consumable verdict.
- Call `main()` at module end (matches emit.ts:154 / score.ts:149).

**2. CREATE `scripts/blueprint-lab/.gitignore`** — one line `results/` (mirrors reviewer-lab, whose `results/` is gitignored per its README). Keeps generated `run.json`/`score.json`/`report.md`/`*.emit.md` out of git.

**3. CREATE `scripts/blueprint-lab/README.md`** — docs mirroring `reviewer-lab/README.md`: what the lab measures (DiffContract emission fidelity vs real landed diffs), the three modules, a `## Run` section (`bun scripts/blueprint-lab/lab.ts`, subset by id, `SKIP_EMIT=1` to reuse, `BLUEPRINT_MODEL`/`BLUEPRINT_EFFORT` overrides passed through to emit.ts:61-62), a `## GATE` section listing the four thresholds and their env vars, and a MODEL-FIDELITY note (measure at sonnet/medium — the daemon's real config, per `emit.ts:61`).
