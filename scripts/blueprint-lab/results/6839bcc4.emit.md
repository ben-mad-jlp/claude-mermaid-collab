## Implementation Blueprint: Throttled archival sweep pass (chunked + yielding, off the loop)

### Context / gap

The prior epic (`ba92cb68`, commit `63de9257`) shipped the **archive storage layer**: the nullable `archivedAt` column (`src/services/todo-store.ts:644`), the partial hot index `idx_todos_hot ... WHERE archivedAt IS NULL` (`:645`), and archive-excluding hot reads in `listTodos`/`listTodosChunked` (`:1381-1382`, `:1421-1422`). **Nothing stamps `archivedAt`** — so the hot working set never actually shrinks and the whole layer is inert.

This leaf adds the **producer**: a sweep that stamps `archivedAt = now` on long-terminal (`done`/`dropped`) rows so they fall out of hot reads. It must be *throttled* (bounded rows per pass), *chunked* (small batches), and *yield off the loop* between batches — unlike the existing `sweepTerminalBucketChildren` (`todo-store.ts:2704`), which loads every bucket via a synchronous `listTodos` monolith and writes inline under a single `withLock` with **no yield**, starving the shared HTTP+daemon event loop (the exact problem `loop-yield.ts` and `listTodosChunked` were built to solve).

### Change shape

**1. `src/services/todo-store.ts`** — add, near the existing sweep (after `sweepTerminalBucketChildren`, ~line 2721):

```ts
/** Terminal (done/dropped) rows older than this are archived out of the hot set. */
export const ARCHIVE_TERMINAL_AFTER_MS = 14 * 24 * 60 * 60 * 1000;
/** Rows stamped per lock acquisition before ceding the loop. */
export const ARCHIVE_SWEEP_BATCH = 500;
/** Max rows a single pass will archive; a larger backlog drains across passes (throttle). */
export const ARCHIVE_SWEEP_MAX_PER_PASS = 5000;

/** Idempotent, chunked, event-loop-friendly archival sweep. Stamps `archivedAt = now`
 *  on terminal (status IN ('done','dropped')) rows whose updatedAt is older than the
 *  cutoff and that are not already archived, so they drop out of hot listTodos reads.
 *  Each batch takes the project lock briefly for one bounded UPDATE, then RELEASES it
 *  and yields OUTSIDE the lock — so HTTP handlers and other project writes interleave.
 *  Returns the archived ids. Filtering on `archivedAt IS NULL` makes re-runs no-ops. */
export async function sweepArchiveTerminalTodos(
  project: string,
  opts: { olderThanMs?: number; batchSize?: number; maxPerPass?: number; yieldFn?: () => Promise<void> } = {},
): Promise<string[]> {
  const olderThanMs = opts.olderThanMs ?? ARCHIVE_TERMINAL_AFTER_MS;
  const batchSize = Math.max(opts.batchSize ?? ARCHIVE_SWEEP_BATCH, 1);
  const maxPerPass = Math.max(opts.maxPerPass ?? ARCHIVE_SWEEP_MAX_PER_PASS, 1);
  const doYield = opts.yieldFn ?? (async () => {});
  const cutoff = new Date(Date.now() - olderThanMs).toISOString();
  const archived: string[] = [];
  for (;;) {
    const remaining = maxPerPass - archived.length;
    if (remaining <= 0) break;
    const limit = Math.min(batchSize, remaining);
    // Short-lived lock: SELECT a bounded page + stamp it, then release before yielding.
    const ids = await withLock(project, () => {
      const db = openDb(project);
      const rows = db.query(
        `SELECT id FROM todos
           WHERE status IN ('done','dropped') AND archivedAt IS NULL AND updatedAt < ?
           ORDER BY updatedAt ASC LIMIT ?`,
      ).all(cutoff, limit) as { id: string }[];
      const now = Date.now();
      for (const r of rows) {
        db.prepare(`UPDATE todos SET archivedAt = ? WHERE id = ?`).run(now, r.id);
      }
      return rows.map((r) => r.id);
    });
    archived.push(...ids);
    if (ids.length < limit) break;   // page not full ⇒ no more candidates
    await doYield();                 // cede the loop between batches (lock NOT held)
  }
  return archived;
}
```

Key correctness points a reviewer can verify: writes stamp `archivedAt` only (they do **not** touch `updatedAt`, and the sweep intentionally does **not** change `status`); the `archivedAt IS NULL` predicate guarantees idempotency and forward progress (each batch shrinks the candidate set, so a plain `LIMIT` needs no keyset cursor); the yield sits *outside* `withLock` so no lock and no open statement is held across a macrotask.

// 3h. ARCHIVAL SWEEP: stamp archivedAt on long-terminal rows so hot reads shrink.
// Chunked + throttled + yields off the loop (see sweepArchiveTerminalTodos).
await yieldToLoop();
try {
  await sweepArchiveTerminalTodos(project, { yieldFn: yieldToLoop });
} catch (err) {
  console.warn(`[reconcile-pass] archival sweep failed for ${project}:`,

}

### Acceptance criteria (positive, citable)

1. `sweepArchiveTerminalTodos` is exported from `src/services/todo-store.ts` and stamps `archivedAt` on `done`/`dropped` rows older than the cutoff via a bounded `LIMIT`ed UPDATE.
2. `ARCHIVE_TERMINAL_AFTER_MS` (and `ARCHIVE_SWEEP_BATCH`) constants exist in `src/services/todo-store.ts`.
3. The sweep's `await doYield()` call sits between batches and *outside* the `withLock` body in `src/services/todo-store.ts`.
4. `src/services/reconcile-pass.ts` imports and calls `sweepArchiveTerminalTodos(project, { yieldFn: yieldToLoop })` inside a `try/catch` in the reconcile pass.
5. Test `archives terminal rows older than cutoff (excluded from hot listTodos, present under onlyArchived)` passes.
6. Test `leaves live and recently-terminal rows hot` passes.
7. Test `is idempotent (second run archives nothing)` passes.
8. Test `chunks in batches and yields between them` passes (small `batchSize`, injected `yieldFn` spy asserted called ≥1 across a multi-batch candidate set).

- Not modifying `sweepTerminalBucketChildren` (it sets `status='dropped'`; those rows become archival candidates here — the two layers compose).
