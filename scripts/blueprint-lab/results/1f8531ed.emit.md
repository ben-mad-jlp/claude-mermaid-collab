**Seed the work-graph** (adapting `landed-epic-sweep.test.ts:39` `seedConvergedEpic`, but with REAL branches):

This is the sole branch `gcEpicBranches` (`landed-epic-sweep.ts:144`, called at `sweep-measurement.ts:88` step 3) will actually delete.

- **Divergence + flag E2** — epic `E2`, then create a real ahead-by-1 branch: `git(['checkout','-b',epicBranchName(E2.id)])`, commit one change, `git(['checkout','master'])`. Stamp it: `stampEpicLandedAt(project, E2.id, new Date(0).toISOString())`. With `landedAt` set, **no done [LAND] child**, and real `ahead===1`, `findLandedAtDivergence` (`invariant-check.ts:159`, via `sweep-measurement.ts:79` step 2) flags `E2.id`; the same ahead>0 branch is left intact and pushed to `gcFlagged` by `gcEpicBranches` (`landed-epic-sweep.ts:162`).

- **Promotion/starvation session `sq`** — mission `M3` under session `sq`, approved (no `awaitingApprovalSince`), queued via `enqueueMission(project, M3.id)` (`mission-store.ts:531` → sets `queuePos`, `active=0`), with `sq` holding **no** active mission. `promoteQueuedMissions` (`mission-store.ts:562`, `sweep-measurement.ts:66` step 1) promotes `M3`.

**Run the primitive TWICE with defaults** (no `opts` → real probe/runner, the live path):
```ts
const snap1 = runSweepMeasurement(project);
const snap2 = runSweepMeasurement(project);
```

**Pass-1 assertions (`snap1`)** — one `check(...)` each:
- `snap1.promoted.includes(M3.id)` — queue promotion fired.
- `snap1.gcDeleted.includes(epicBranchName(E1.id))` — real branch deleted (step 3).
- `snap1.gcFlagged.includes(E2.id)` — ahead>0 branch flagged, not deleted.
- `snap1.landedAtDivergence.ids.includes(E2.id)` && `snap1.landedAtDivergence.count >= 1` — divergence detected while E2 branch still ahead (step 2 precedes step 3).
- `!snap1.fullyOnMasterBranchesRemaining.includes(epicBranchName(E1.id))` — the post-GC re-probe (step 4, `sweep-measurement.ts:96`) sees E1's branch gone. *(Positive form: also assert this array is a proper subset — E1 branch absent because deleted.)*
- `snap1.sessionsZeroActiveWithQueuedApproved.length === 0` — the composed dependency order (promotion before starvation check, `sweep-measurement.ts:50` doc) self-healed `sq` to zero.
- **Recovery log**: `.collab/pruned-branches-recovery.md` (`landed-epic-sweep.ts:129`) exists and its contents include `epicBranchName(E1.id)` — proves the live delete captured a recoverable tip SHA.

**Pass-2 idempotence assertions (`snap2` vs `snap1`)**:
- `snap2.promoted.length === 0` — M3 already active, nothing to re-promote.
- `snap2.gcDeleted.length === 0` — E1 branch already gone.
- `snap2.gcFlagged.includes(E2.id)` — flagging is read-only and stable across runs.
- `JSON.stringify(snap2.landedAtDivergence) === JSON.stringify(snap1.landedAtDivergence)` — divergence surface byte-stable.
- `JSON.stringify(snap2.fullyOnMasterBranchesRemaining) === JSON.stringify(snap1.fullyOnMasterBranchesRemaining)` — re-probe surface byte-stable. This pair is the concrete realization of the primitive's "diff two consecutive runs" idempotence contract.

**Cleanup** (`finally`, mirroring `landed-epic-sweep.test.ts:22-30`): `_closeProject(project)`, `_resetMissionDbCache(project)`, `_closeLedgerDb()`, `delete process.env.MERMAID_SUPERVISOR_DIR`, `rmSync(project,{recursive,force})` and the supervisor temp dir. Finish with the `smoke-pool-live.ts:166` summary line + `process.exit(fail===0?0:1)`.
