## Blueprint — Emission runner for blueprint-lab

### Goal
Add the **emission stage** of `blueprint-lab`: a runner that, for each mined corpus case, runs the **real blueprint node** (same `claude -p` invocation shape the daemon executor uses, at the **daemon's real model** — sonnet/medium by default) and captures a **v2 `DiffContract`** it emits, parsed with the shipped `parseDiffContract`. This is the blueprint-side analog of `scripts/reviewer-lab/run.ts`. Scoring the emitted contract against the corpus's real diff is a *later* leaf — this leaf only emits + records.

### Context (real files/symbols this builds on)
- `scripts/blueprint-lab/corpus.ts` — exports `CORPUS: CorpusCase[]` (`corpus.ts:171`), each with `diff.baseSha` (`corpus.ts:28`), `leafKind`, and `spec.{title,description}` (`corpus.ts:21-26`). 48 real landed leaves, kinds feature/fix/test.
- `src/services/diff-contract.ts` — `parseDiffContract(...sources)` (fail-safe, reads LAST ```json fence, `diff-contract.ts:162`), `validateContractForKind(contract, leafKind)` (`diff-contract.ts:99`), `DiffContract`/`DiffLeafKind` types.
- `src/agent/node-invoker.ts` — `buildNodeArgv(spec: NodeSpec)` (`node-invoker.ts:307`) builds the exact executor argv; prompt goes on **stdin**, not positional (`node-invoker.ts:337`).
- `src/services/leaf-executor.ts` — `buildNodePrompt('blueprint', leaf)` (`leaf-executor.ts:814-865`) carries the positive/citable-criteria discipline but a **v1** trailing manifest. The runner needs a **v2** trailing fence, so it owns a local `buildEmitPrompt` (no shipped v2 node mode exists yet — verified: no `schemaVersion 2` prompt builder anywhere).
- Pattern reference: `scripts/reviewer-lab/run.ts` (`runReviewNode` at `run.ts:89`, `main` concurrency-8 worker pool at `run.ts:169`, `results/` + `run.json`).

### Changes

**1. Create `scripts/blueprint-lab/run.ts`** (new file) with these symbols:

- `buildEmitPrompt(leaf: { id: string; title: string; description: string }): string` — self-contained blueprint-node prompt mirroring the real node's positive/citable-criteria discipline (from `leaf-executor.ts:816-834`), but whose **trailing fence is the v2 DiffContract shape** (`schemaVersion: 2`, `leafKind`, `requirements[]` union, `outOfScope`) exactly as in `diff-contract.ts:79-92` and the `DiffRequirement` union (`diff-contract.ts:47-50`). Instructs "FINISH with EXACTLY ONE trailing ```json fence, the LAST fence in the reply". Reply-only (no file write — the runner reads the node's final message).

- `setupBaseCheckout(c: CorpusCase): { dir: string; cleanup: () => void }` — creates a detached checkout of this repo at `c.diff.baseSha` via `git worktree add --detach <tmpdir> <baseSha>` (confirmed supported), so the node reads the **real code at the base the leaf started from** and can cite real symbols. `cleanup` runs `git worktree remove --force <dir>`.

- `runEmitNode(cwd: string, prompt: string): Promise<{ text: string }>` — builds a `NodeSpec` (`model: process.env.BLUEPRINT_MODEL || 'sonnet'`, `effort: process.env.BLUEPRINT_EFFORT || 'medium'`, `allowedTools: 'Read Grep Glob Bash'`, `permissionMode: 'bypassPermissions'`, `strictMcpConfig: true`), calls `buildNodeArgv(spec)`, spawns it, writes `prompt` to stdin, and extracts the final `{"type":"result"}` `.result` text from the stream-json — same extraction as `reviewer-lab/run.ts:111-123`.

- `emitOne(c: CorpusCase)` — `setupBaseCheckout` → `buildEmitPrompt` → `runEmitNode` → `parseDiffContract(text)` → `validateContractForKind(contract, contract.leafKind)`; writes the raw reply to `results/<id>.emit.md`; returns a record `{ id, expectedKind: c.leafKind, emittedKind, parsed: boolean, underspecified: boolean, missingField? }`; always calls `cleanup` (finally).

- `main()` — optional `process.argv.slice(2)` id-subset filter (like `run.ts:170-171`), a fixed-concurrency worker pool (default 4 — LLM-bound), prints a per-case `✓/✗` line, and writes `results/run.json` with `{ total, parsedCount, wellFormedCount, results }`.

**2. Edit `scripts/blueprint-lab/.gitignore`** — add `results/` so raw emissions + `run.json` stay untracked (matches `reviewer-lab/.gitignore`).

**3. Create `scripts/blueprint-lab/README.md`** — document `bun scripts/blueprint-lab/run.ts [ids…]`, the `BLUEPRINT_MODEL`/`BLUEPRINT_EFFORT` overrides, model-fidelity note (measure at the daemon's real sonnet/medium), and that diff-scoring is a later stage.

### Acceptance criteria (positive, citable)
1. `scripts/blueprint-lab/run.ts` defines `buildEmitPrompt` whose trailing fenced JSON template uses `"schemaVersion": 2` and names `leafKind`/`requirements`/`outOfScope`.
2. `run.ts` defines `runEmitNode` that calls `buildNodeArgv` imported from `../../src/agent/node-invoker` and defaults `model` to `process.env.BLUEPRINT_MODEL || 'sonnet'`.
3. `run.ts` defines `setupBaseCheckout` that shells `git worktree add --detach` at `c.diff.baseSha`.
4. `run.ts` defines `emitOne` that calls `parseDiffContract` and `validateContractForKind` imported from `../../src/services/diff-contract`.
5. `run.ts` defines `main` that filters `CORPUS` (imported from `./corpus`) by an id subset and writes `results/run.json`.
6. `scripts/blueprint-lab/.gitignore` contains a `results/` line.

### Non-goals (prose, not criteria)
- Scoring the emitted contract against the corpus case's real `diff.touchedFiles`/`changedSymbols` — that's a separate downstream leaf (per `corpus.ts:6-8`).
- Adding a v2 mode to `buildNodePrompt`/`leaf-executor.ts` — the runner owns its emit prompt locally to keep this leaf uncoupled.
