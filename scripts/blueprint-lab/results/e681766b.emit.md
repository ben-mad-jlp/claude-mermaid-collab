## Context

`scripts/smoke-conductor-pin-live.ts` already drives the full two-mission conductor-pin scenario against a deployed server (`--base-url`), driving the criterion verdict over the live `/mcp` endpoint (commit `39a71973`). It emits results to **stdout only** — a `  ✅/❌ <name> — <detail>` line per check, a `📄 transcript: <json>` line, and a `✅ ALL PASS — N passed, M failed` summary — then `process.exit(fail === 0 ? 0 : 1)`. Nothing is persisted, so a conductor/steward marking a criterion `met` (which stores `evidence` + `evidencePaths`, `src/services/mission-store.ts:544`) has no citable artifact.

This leaf adds a **live evidence harness**: a small additive script that runs that smoke against the deployed server, parses its output into a structured verdict, and writes a durable JSON evidence file whose path can be cited. It does **not** modify the existing smoke (keeps the working live driver untouched — see `[[project_daemon_builder_untrustworthy]]`), and it is decoupled: it spawns the smoke as a subprocess and parses stdout, so the two scripts share no mutable state.

```ts
export interface PinCheck { name: string; ok: boolean; detail: string; }
export interface PinEvidence {
  harness: 'conductor-pin-live';
  baseUrl: string;
  capturedAt: string;      // ISO — new Date().toISOString() (allowed here; plain bun script)
  exitCode: number;
  passed: number;
  failed: number;
  ok: boolean;             // exitCode === 0 && failed === 0 && checks non-empty
  checks: PinCheck[];
  transcript: unknown[];   // parsed from the "📄 transcript:" line
  rawOutput: string;
}

// PURE — no I/O, no subprocess. The unit-testable seam.
export function parseSmokeOutput(stdout: string, exitCode: number, baseUrl: string): PinEvidence { … }

// resolve --base-url / MERMAID_SMOKE_BASE_URL; throw if absent (this harness is LIVE-ONLY).
export function resolveBaseUrl(argv: string[]): string { … }

// spawn the smoke, capture, parse, write --out, print path. Returns process exit code.
export async function main(argv: string[]): Promise<number> { … }

if (import.meta.main) {
  main(process.argv.slice(2)).then((code) => process.exit(code));
}
```

Parsing rules (match the exact strings the smoke prints, `smoke-conductor-pin-live.ts:57,374,375`):
- **checks**: each line matching `/^ {2}(✅|❌) (.+)$/`; split the remainder on the first ` — ` into `{ name, detail }`; `ok = emoji === '✅'`.
- **transcript**: the line beginning `📄 transcript: ` → `JSON.parse(line.slice('📄 transcript: '.length))`; default `[]` if absent/unparseable.
- **summary**: `/(\d+) passed, (\d+) failed/` → `passed`, `failed`.
- `ok = exitCode === 0 && failed === 0 && checks.length > 0`.

`main()` behavior:
- `resolveBaseUrl` reads `--base-url <url>`, `--base-url=<url>`, or `MERMAID_SMOKE_BASE_URL`; **throws** a clear error when none is set (live-only — no in-process fallback, since the point is evidence against the *deployed* server).
- Spawn `Bun.spawn(['bun', 'run', 'scripts/smoke-conductor-pin-live.ts', '--base-url', baseUrl], { cwd: <repo root via import.meta.dir/..>, stdout: 'pipe', stderr: 'pipe' })`; await exit; read stdout+stderr text.
- `const ev = parseSmokeOutput(stdout + stderr, exitCode, baseUrl)`.
- Resolve `--out <path>` (default `./conductor-pin-evidence.json` in cwd); `writeFileSync(outPath, JSON.stringify(ev, null, 2))`.
- `console.log` the resolved evidence path and a one-line verdict (`ok ? '✅' : '❌'`, `passed/failed`).
- Return `exitCode` (mirror the smoke's own exit).

**Create `scripts/__tests__/conductor-pin-evidence.test.ts`** — `bun:test` (matches the runner used by `scripts/__tests__/parent-epic-under-mission.test.ts`; run directly via `bun test <file>`). Fully offline — feed canned stdout fixtures to the pure exports, no live server:
- A PASS fixture (a few `  ✅ …` lines + `📄 transcript: [{"tick":1,...}]` + `✅ ALL PASS — 24 passed, 0 failed`) → assert `checks` parsed with names/details, `transcript` length, `passed===24`, `failed===0`, `ok===true`.
- A FAILURE fixture (mix of `✅/❌`, `❌ FAILURES — 20 passed, 4 failed`, exitCode `1`) → assert `failed===4`, `ok===false`, and at least one `check.ok===false`.
- `resolveBaseUrl` throws when argv/env give no URL; returns the value from `--base-url`.

## Acceptance criteria (positive, citable)

1. `scripts/conductor-pin-evidence.ts:parseSmokeOutput` returns a `PinEvidence` whose `checks[]` are extracted from `  ✅/❌` lines, `transcript` from the `📄 transcript:` line, and `passed`/`failed` from the summary line.
2. `scripts/conductor-pin-evidence.ts:main` spawns `smoke-conductor-pin-live.ts` with the forwarded `--base-url` and calls `writeFileSync` to persist the evidence JSON to the `--out` path (default `conductor-pin-evidence.json`), logging that path.
3. `scripts/conductor-pin-evidence.ts:resolveBaseUrl` throws when no base URL is provided (live-only guard).
4. `scripts/conductor-pin-evidence.ts` guards execution behind `if (import.meta.main)` so the module is importable by the test without running.
5. `scripts/__tests__/conductor-pin-evidence.test.ts` contains a test asserting `parseSmokeOutput` extracts checks/transcript/summary from a PASS fixture, and one asserting `ok===false` on a FAILURE fixture.
