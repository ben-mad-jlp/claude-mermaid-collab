# Blueprint — `queuePos` column + `enqueueMission`/`promoteMission` store primitives

## Context

`src/services/mission-store.ts` owns the durable `.collab/mission.db` loop-control state for `[MISSION]` nodes. Today missions have no ordering — `selectConductorMission` (`mission-store.ts:531`) picks the next mission by a derived status-rank tiebreak, with no human/steward-controllable queue. This leaf adds the **storage substrate** for an explicit mission queue: one nullable `queuePos` column plus two write primitives (`enqueueMission`, `promoteMission`). It is deliberately store-only — no conductor/selection wiring, no MCP tool, no UI. Those are downstream leaves.

### Queue semantics (locked for this leaf)
- `queuePos: number | null` on the mission row. **`null` = not in the queue.** A non-null value orders queued missions **ascending — lower `queuePos` = closer to the front** (served sooner).
- **`enqueueMission`** appends to the **back**: if already queued (`queuePos != null`) it is a no-op and returns the row unchanged (idempotent); otherwise it assigns `MAX(queuePos)+1` over currently-queued rows, defaulting to `0` on an empty queue.
- **`promoteMission`** jumps to the **front**: it sets `queuePos = MIN(queuePos)-1` over the *other* queued rows (default `0` when it is the only/first queued mission). Always writes; re-promoting a front mission keeps it at the front. Enqueues a not-yet-queued mission as a side effect (front insert).
- Both bump `updatedAt` and throw `mission not found: <id>` when the row is absent (same contract as `setMissionActive` at `mission-store.ts:437-442`).

```ts
/** Append a mission to the BACK of the steward queue. Idempotent: a mission already in
 *  the queue (queuePos != null) is returned unchanged. A fresh enqueue gets MAX(queuePos)+1
 *  over currently-queued rows (0 on an empty queue) — queuePos orders ascending, front-first. */
export function enqueueMission(project: string, todoId: string): MissionRow {
  const m = getMission(project, todoId);
  if (!m) throw new Error(`mission not found: ${todoId}`);
  if (m.queuePos != null) return m; // already queued — no-op
  const db = openDb(project);
  const row = db.query('SELECT MAX(queuePos) AS maxPos FROM mission WHERE queuePos IS NOT NULL').get() as { maxPos: number | null };
  const nextPos = row.maxPos == null ? 0 : row.maxPos + 1;
  db.prepare('UPDATE mission SET queuePos = ?, updatedAt = ? WHERE todoId = ?').run(nextPos, nowMs(), todoId);
  return getMission(project, todoId)!;
}

/** Move a mission to the FRONT of the steward queue: queuePos = MIN(queuePos)-1 over the
 *  OTHER queued rows (0 when it is the only/first queued mission). Enqueues a not-yet-queued
 *  mission as a front insert. Always writes; re-promoting a front mission keeps it in front. */
export function promoteMission(project: string, todoId: string): MissionRow {
  const m = getMission(project, todoId);
  if (!m) throw new Error(`mission not found: ${todoId}`);
  const db = openDb(project);
  const row = db.query('SELECT MIN(queuePos) AS minPos FROM mission WHERE queuePos IS NOT NULL AND todoId != ?').get(todoId) as { minPos: number | null };
  const frontPos = row.minPos == null ? 0 : row.minPos - 1;
  db.prepare('UPDATE mission SET queuePos = ?, updatedAt = ? WHERE todoId = ?').run(frontPos, nowMs(), todoId);
  return getMission(project, todoId)!;
}
```
Note: `getMission` on the return path re-derives status (consistent with `setMissionActive`'s callers and `upsertMission`'s `return getMission(...)!` at `mission-store.ts:317`), so the returned row carries the fresh `queuePos` plus derived `status`.

### 2. `src/services/__tests__/mission-store.test.ts`

Add the two primitives to the import block (`mission-store.test.ts:9-15`) and a new `describe('mission-store: queue', ...)` block using the existing `makeMissionNode` helper (`mission-store.test.ts:31`). Tests (bun:sqlite runner — see `project_dual_test_runner`):

- **`enqueueMission assigns queuePos 0 to the first mission, 1 to the next`** — enqueue two missions; assert `queuePos` values `0` then `1`.
- **`enqueueMission is idempotent`** — enqueue a mission twice; assert the second call returns the same `queuePos` (`0`, unchanged).
- **`promoteMission moves a back mission to the front`** — enqueue m1 (pos 0) and m2 (pos 1), `promoteMission(m2)`; assert m2's `queuePos` (`-1`) is `< ` m1's (`0`).
- **`promoteMission enqueues a not-yet-queued mission at the front`** — enqueue m1 (pos 0), then `promoteMission(m2)` where m2 was never enqueued; assert m2's `queuePos` is `< ` m1's and non-null.
- **`enqueueMission throws for an unknown mission id`** — assert `enqueueMission(project, 'nope')` throws `/mission not found/`.
- **`queuePos defaults to null for a freshly upserted mission`** — `upsertMission` then `getMission`; assert `queuePos` is `null`.
