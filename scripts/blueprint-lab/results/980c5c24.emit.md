## Blueprint: Per-instance CDP port and collision-free tab persistence

### Problem

Two `mermaid-collab` server instances on one host (the documented `PORT=0` multi-instance flow, `docs/multi-instance-setup.md`) collide on two shared, host-global resources:

1. **CDP port** — `CDP_PORT` in `src/config.ts:165-168` defaults to a fixed `9333`. Each instance in `owned-chrome`/`streamed-panel` mode spawns its own Chrome on `--remote-debugging-port=${cdpPort}` (`src/services/chrome-manager.ts:119`). A second instance either fails to bind or — worse — `ChromeManager.waitForCdp` (`chrome-manager.ts:129-146`) polls `/json/version` on `9333`, sees the *first* instance's live Chrome, and falsely reports "ready", so both instances drive the same browser.

2. **Tab persist file** — `TABS_PERSIST_FILE = '/tmp/.mermaid-collab-tabs.json'` (`src/services/cdp-session.ts:79`) is a single host-global path. `persistTabRegistry` (line 84-85) *overwrites* it, and `closePersistedTabs(port)` (line 89-98, called at every server boot in `src/server.ts:68-69`) reads it and closes **every** target listed — so instance B's startup wipes and closes instance A's tabs.

The in-process `tabRegistry` Map and `tabRegistryKey` (`cdp-session.ts:25-34`) are already project-scoped (covered by `src/services/__tests__/cdp-session.registry-key.test.ts`) and need no change; the collision is in the two host-global resources above.

### Change shape

Derive both resources deterministically from the same `(project, session)` identity the discovery layer already uses (`deriveSessionId`, `src/services/instance-discovery.ts:89-91`), so distinct instances get distinct ports/files with no IPC, while the explicit `CDP_PORT` env override (Electron sidecar) is preserved.

**1. `src/config.ts`** — add per-instance CDP port derivation:
- `import { createHash } from 'node:crypto';`
- Add exported `deriveCdpPort(project: string, session: string): number` — sha1 of `project + '\0' + session`, mapped into `9333 + (digest.readUInt16BE(0) % 1000)` (range 9333–10332, wide enough that distinct instances rarely alias).
- Rewrite the `CDP_PORT` IIFE (lines 165-168): if `process.env.CDP_PORT` is set and numeric, honor it verbatim (existing Electron-sidecar override); otherwise return `deriveCdpPort(MERMAID_PROJECT, MERMAID_SESSION)` (both already declared at `config.ts:89` and `:95`). Update the doc comment at `config.ts:159-164` to describe the per-instance default.

**2. `src/services/cdp-session.ts`** — per-port tab persist path:
- Add `import { tmpdir } from 'node:os';` and `import { join } from 'node:path';`.
- Replace the constant at line 79 with an exported pure function `export function tabsPersistFile(port: number): string` returning `join(tmpdir(), \`.mermaid-collab-tabs-${port}.json\`)`.
- `persistTabRegistry` (line 81-87) writes to `tabsPersistFile(CDP_PORT)`.
- `closePersistedTabs(port)` (line 89-98) reads/unlinks `tabsPersistFile(port)` (parameter already threaded from `server.ts:68-69`, which passes `CDP_PORT`).

**3. `src/services/__tests__/cdp-port.test.ts`** (new) — assert `deriveCdpPort` gives distinct ports for the two multi-instance-doc sessions (`feature-x` vs `feature-y` under one project), and that a set CDP_PORT env value is not consulted by `deriveCdpPort` (pure of env).

**4. `src/services/__tests__/cdp-session.persist-isolation.test.ts`** (new) — assert `tabsPersistFile(9333) !== tabsPersistFile(9334)` and that the returned path is under `os.tmpdir()`.

### Notes / edges
- Deterministic derivation (not `PORT=0`-style ephemeral) is required: the sidecar, CLI, and hooks must agree on the port without a handshake. The `% 1000` range makes accidental aliasing between two live instances negligible; `ChromeManager.waitForCdp`'s existing false-positive on a shared port is thereby avoided in practice, not by new logic.
- No change to `tabRegistryKey`, the in-process `tabRegistry` Map, `withCDPSession`, `ensureTab`, or the Electron `runtimeElectronTarget` path.

### Acceptance criteria
1. `src/config.ts` exports `deriveCdpPort(project, session)` returning a port ≥ 9333 within the derived range — reviewer can point at its definition.
2. `src/config.ts` `CDP_PORT` returns `deriveCdpPort(MERMAID_PROJECT, MERMAID_SESSION)` when `process.env.CDP_PORT` is unset, and returns the parsed env value when set — pointable at the rewritten IIFE.
3. `src/services/cdp-session.ts` exports `tabsPersistFile(port)` producing a per-port path under `os.tmpdir()`.
4. `persistTabRegistry` and `closePersistedTabs` in `cdp-session.ts` reference `tabsPersistFile(...)` (the shared `/tmp/.mermaid-collab-tabs.json` literal is gone).
5. `cdp-port.test.ts` names a passing test proving two distinct sessions yield two distinct `deriveCdpPort` values.
6. `cdp-session.persist-isolation.test.ts` names a passing test proving `tabsPersistFile` returns distinct paths for distinct ports.
