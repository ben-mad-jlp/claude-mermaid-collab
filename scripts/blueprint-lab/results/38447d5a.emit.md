## Blueprint: Orchestration-node registry

### Problem being solved
Three server-side orchestration nodes each hand-roll the *same* three-part defaults + resolution triad, with copy-pasted constants:

- `src/mcp/tools/mission-forge.ts:251-253` — `FORGE_DEFAULT_MODEL='opus'`, `FORGE_DEFAULT_EFFORT='high'`, `FORGE_ALLOWED_TOOLS='Read Grep Glob Bash'`
- `src/services/conductor-pass.ts:27-40` — `CONDUCTOR_DEFAULT_MODEL='opus'`, `CONDUCTOR_DEFAULT_EFFORT='high'`, `CONDUCTOR_ALLOWED_TOOLS=[…].join(' ')`
- `src/mcp/tools/mission-planner.ts:22-26` — `PLANNER_DEFAULT_MODEL='opus'`, `PLANNER_DEFAULT_EFFORT='high'`, `PLANNER_ALLOWED_TOOLS='Read Grep Glob Bash …'`

Each then repeats the identical resolution shape (`resolveNodeProvider` → `resolveNodeModel(project,kind,provider,DEFAULT_MODEL)` → `listNodeProfileOverrides(project)[kind]?.effort ?? DEFAULT_EFFORT`). There is no single place that says "these are the orchestration node kinds and their defaults." This leaf introduces that registry and repoints the three nodes at it.

### Change shape

**1. New file `src/services/orchestration-node-registry.ts`** — the single source of truth.

```ts
import type { EffortLevel } from '../agent/contracts.js';

/** The server-side ORCHESTRATION node kinds (mission-driving nodes, distinct from the
 *  leaf-executor build nodes). Each is high-judgment → opus/high by default, and each is
 *  per-project configurable via node_profile_override(project, <kind>, model, effort). */
export type OrchestrationNodeKind = 'forge' | 'conductor' | 'planner';

export interface OrchestrationNodeDefaults {
  /** The node_profile_override key + resolveNodeModel/resolveNodeProvider `kind`. */
  kind: OrchestrationNodeKind;
  /** Claude-family default model (opus). resolveNodeModel swaps this for the grok default
   *  when the node's provider resolves to a grok lane. */
  model: string;
  /** Default reasoning effort when no per-project override is set. */
  effort: EffortLevel;
  /** The node's tool allowlist (also drives the MCP-forced-claude provider guard). */
  allowedTools: string;
}

export const ORCHESTRATION_NODE_REGISTRY: Record<OrchestrationNodeKind, OrchestrationNodeDefaults> = {
  forge: {
    kind: 'forge',
    model: 'opus',
    effort: 'high',
    allowedTools: 'Read Grep Glob Bash',
  },
  conductor: {
    kind: 'conductor',
    model: 'opus',
    effort: 'high',
    allowedTools: [
      'Read', 'Grep', 'Glob', 'Bash',
      'mcp__mermaid__get_mission', 'mcp__mermaid__get_task_graph', 'mcp__mermaid__get_todo',
      'mcp__mermaid__plan_mission_criterion', 'mcp__mermaid__file_to_bucket',
      'mcp__mermaid__set_mission_criterion', 'mcp__mermaid__add_mission_criterion',
      'mcp__mermaid__escalation_list', 'mcp__mermaid__epic_land_readiness', 'mcp__mermaid__verify_epic',
      'mcp__mermaid__land_epic', 'mcp__mermaid__consult_grok',
    ].join(' '),
  },
  planner: {
    kind: 'planner',
    model: 'opus',
    effort: 'high',
    allowedTools: 'Read Grep Glob Bash mcp__mermaid__get_task_graph mcp__mermaid__get_todo mcp__mermaid__get_mission',
  },
};

/** The defaults for one orchestration node kind (single source of truth for its
 *  model/effort/allowedTools). Throws on an unknown kind. */
export function orchestrationNodeDefaults(kind: OrchestrationNodeKind): OrchestrationNodeDefaults {
  const d = ORCHESTRATION_NODE_REGISTRY[kind];
  if (!d) throw new Error(`orchestrationNodeDefaults: unknown orchestration node kind "${kind}"`);
  return d;
}
```

*(This module has no runtime imports beyond the `EffortLevel` type, so it introduces no import cycle with `node-provider.ts`, `conductor-pass.ts`, or the mcp tools.)*
