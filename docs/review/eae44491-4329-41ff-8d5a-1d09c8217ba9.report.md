## Findings

**1. (Material) Verdict bar ignores wedged/unknown sessions — green "All clear" over a stuck session.** `selectVerdict` (`ui/src/lib/freshnessSelectors.ts:41-60`) folds only connection-state + `openEscalations.length`; it never reads `sessionSummaries`. A wedged/unknown session with no open escalation yields `tone:'clear'`/`"All clear"` (green) in Zone 0 — while `ZenMode` simultaneously promotes that same wedge into the FocusCard. The across-the-room signal ("the only thing the compulsive glance must read") shows calm green while the card shows STUCK. This contradicts the design's Zone-0 wireframe ("▲ 1 session stuck · 2 more need you"), guarantee #2, the `selectVerdict` spec ("folds needs-you count + **any wedge** + connection state"), and the vision's core thesis ("a serene panel that is wrong once is worse than no panel"). Filed: `899f33a7`.

**2. (Minor) `VerdictTone 'attention'` (amber) is defined but never emitted.** `selectVerdict` only returns `disconnected | clear | urgent`; the graded amber "come closer" signal never reaches Zone 0 (verdict jumps green→red). Coupled to finding #1. Filed: `416e00bb`.

## Conclusion
All eight named spec elements are implemented correctly and the backend loop is genuinely scheduled and live. That is precisely the "serene panel that is wrong" the design forbids, so it is a material gap.
